# ✅ Function Calls (Step-by-Step)

## Lesson 18: What the CPU Actually Does When You Call a Function

---

# 🎯 Learning Objective

By the end of this lesson, I should be able to answer:

* What is a Function Call?
* Why do functions exist?
* What happens when a function is called?
* How does the CPU execute a function call?
* What is a Stack Frame?
* How are parameters passed?
* How are return values returned?
* Why is the Stack important for function calls?
* What would happen if function calls did not exist?

---

# 🌍 The Fundamental Problem

Imagine writing:

```c
printf("Hello");
printf("Hello");
printf("Hello");
printf("Hello");
printf("Hello");
```

Suppose the same code is needed:

```text
10 Times

100 Times

1000 Times
```

Writing it repeatedly is inefficient.

Engineers invented:

# Functions

A function allows us to:

```text
Write Once

Use Many Times
```

---

## What is a Function?

A function is:

> A reusable block of code that performs a specific task.

Example:

```c
void greet()
{
    printf("Hello");
}
```

---

## Function Call

Using the function:

```c
greet();
```

This is called a:

# Function Call

---

## Real-Life Analogy

Imagine a restaurant.

You:

```text
Place Order
```

Kitchen:

```text
Prepares Food
```

Food:

```text
Returned To You
```

Similarly:

```text
Call Function

↓

Function Executes

↓

Returns Control
```

---

## Example

```c
void greet()
{
    printf("Hello");
}

int main()
{
    greet();
}
```

Execution:

```text
main()

↓

greet()

↓

Print Hello

↓

Return To main()
```

---

## What Actually Happens?

Most beginners think:

```text
greet();

↓

Magic
```

No.

The CPU performs many steps.

---

## Step 1: Function Call Instruction

Suppose:

```c
greet();
```

Compiler may generate:

```assembly
CALL greet
```

Meaning:

```text
Jump To greet()

But Remember

Where To Return
```

---

## Why Remember?

After:

```c
greet();
```

execution must continue at:

```c
printf("Done");
```

Example:

```c
greet();

printf("Done");
```

CPU must remember:

```text
Where execution should continue.
```

---

## Return Address

Before jumping:

CPU stores:

```text
Address Of Next Instruction
```

Example:

```text
100 → CALL greet

101 → printf("Done")
```

CPU saves:

```text
101
```

This is called:

# Return Address

---

## Where Is Return Address Stored?

On the:

# Stack

Example:

```text
Stack

+----------------+
| Return Address |
+----------------+
```

---

## Step 2: Create Stack Frame

Function:

```c
void greet()
{
    int x = 10;
}
```

CPU creates:

# Stack Frame

A Stack Frame stores:

```text
Parameters

Local Variables

Return Address

Saved Registers
```

---

## Example

```c
void greet()
{
    int x = 10;
}
```

Stack:

```text
+----------------+
| Return Address |
+----------------+
| x = 10         |
+----------------+
```

---

## Why Stack Frames Exist

Imagine:

```c
funcA()
{
    funcB();
}
```

and:

```c
funcB()
{
}
```

CPU must remember:

```text
Which Function Called Which
```

Stack Frames solve this problem.

---

## Example

```text
main()

↓

funcA()

↓

funcB()
```

Stack:

```text
+-----------+
| funcB     |
+-----------+
| funcA     |
+-----------+
| main      |
+-----------+
```

---

## Step 3: Execute Function

After Stack Frame creation:

CPU executes instructions inside:

```c
greet()
```

Example:

```c
void greet()
{
    printf("Hello");
}
```

CPU executes:

```text
Print Hello
```

---

## Step 4: Return Statement

Example:

```c
return;
```

or

```c
return 25;
```

Function finishes.

---

## Step 5: Remove Stack Frame

CPU removes:

```text
Local Variables

Parameters

Temporary Data
```

from the stack.

Example:

Before:

```text
+----------------+
| Return Address |
+----------------+
| x              |
+----------------+
```

After:

```text
Removed
```

---

## Step 6: Return To Caller

CPU uses:

```text
Return Address
```

stored earlier.

Example:

```text
Return Address = 101
```

CPU jumps back:

```text
Instruction 101
```

Program continues.

---

## Complete Function Call Flow

```text
CALL Function

↓

Store Return Address

↓

Create Stack Frame

↓

Execute Function

↓

Remove Stack Frame

↓

Return To Caller
```

This happens every function call.

---

## Parameters

Example:

```c
void add(int a, int b)
{
}
```

Call:

```c
add(10, 20);
```

Parameters:

```text
a = 10

b = 20
```

must be passed to the function.

---

## Stack Example

```text
+----------------+
| Return Address |
+----------------+
| a = 10         |
+----------------+
| b = 20         |
+----------------+
```

Function accesses:

```text
a

b
```

from its Stack Frame.

---

## Return Values

Example:

```c
int add()
{
    return 50;
}
```

Question:

> How does 50 return to the caller?

---

## Simple View

CPU places:

```text
50
```

into a special register.

Then returns.

Example:

```text
Return Register

↓

50
```

Caller reads it.

---

## Example

```c
int result = add();
```

Execution:

```text
add()

↓

Return 50

↓

Special Register

↓

result = 50
```

---

## Nested Function Calls

Example:

```text
main()

↓

funcA()

↓

funcB()

↓

funcC()
```

Stack:

```text
+-----------+
| funcC     |
+-----------+
| funcB     |
+-----------+
| funcA     |
+-----------+
| main      |
+-----------+
```

---

## Returning

Functions return in reverse order:

```text
funcC

↓

funcB

↓

funcA

↓

main
```

This is why Stack uses:

# LIFO

Last In First Out

---

## Recursion

Example:

```c
void func()
{
    func();
}
```

Every call creates:

```text
New Stack Frame
```

Example:

```text
func

func

func

func

func
```

Stack keeps growing.

Eventually:

```text
Stack Full
```

Result:

# Stack Overflow

---

## Why Function Calls Are Important

Without function calls:

Programs would contain:

```text
Huge Repeated Code
```

making software:

```text
Hard To Read

Hard To Maintain

Hard To Reuse
```

Functions solve this.


All are functions.

## Common Misconceptions

❌ Function calls create new processes.

❌ Local variables are stored on the Heap.

❌ Return addresses are stored in RAM randomly.

❌ Function calls happen without using the Stack.

❌ Returning from a function means restarting the program.

---

# 🧠 Quick Revision

* Function = reusable block of code.
* Function Call = invoking that block.
* CPU performs:

  * Save Return Address
  * Create Stack Frame
  * Execute Function
  * Destroy Stack Frame
  * Return To Caller
* Stack Frames contain:

  * Parameters
  * Local Variables
  * Return Address
  * Saved Registers
* Return values are usually returned through registers.
* Function calls rely heavily on the Stack.
* Recursion creates multiple Stack Frames.
* Too many Stack Frames can cause Stack Overflow.
