# ✅ Static Variables (Step-by-Step)

## Lesson 19: Why Functions Remember (and How to Make Them)

---

# 🎯 Learning Objective

By the end of this lesson, I should be able to answer:

* What is a Static Variable?
* Why was it invented?
* What problem does it solve?
* How is it different from a normal local variable?
* Where is it stored?
* Why does Runtime initialize it before `main()`?
* When should I use static variables?

---

# 🌍 The Problem

Consider:

```c
void counter()
{
    int count = 0;

    count++;

    printf("%d\n", count);
}
```

Now:

```c
counter();
counter();
counter();
```

What will be printed?

---

## First Call

```text
count = 0

count++

↓

1
```

Print:

```text
1
```

---

## Second Call

Function starts again:

```text
count = 0
```

Again:

```text
count++

↓

1
```

Print:

```text
1
```

---

## Third Call

Again:

```text
count = 0
```

Print:

```text
1
```

Output:

```text
1
1
1
```

---

# Why Does This Happen?

Because:

```c
int count = 0;
```

is a normal local variable.

Local variables live on the:

# Stack

When function ends:

```text
Stack Frame Destroyed

↓

count Disappears
```

Next call creates a fresh variable.

---

# What If We Want It To Remember?

Suppose we want:

```text
1
2
3
```

instead of:

```text
1
1
1
```

We need a variable that:

```text
Survives Function Calls
```

This is exactly why:

# Static Variables

were invented.

---

# What is a Static Variable?

A static variable is:

> A variable that is created only once and keeps its value for the entire lifetime of the program.

---

# Example

```c
void counter()
{
    static int count = 0;

    count++;

    printf("%d\n", count);
}
```

---

# Execution

First call:

```text
count = 0

count++

↓

1
```

Print:

```text
1
```

---

Second call:

```text
count already exists

count = 1

count++

↓

2
```

Print:

```text
2
```

---

Third call:

```text
count = 2

count++

↓

3
```

Print:

```text
3
```

Output:

```text
1
2
3
```

---

# The Big Difference

Normal Variable:

```c
int count = 0;
```

Every function call:

```text
Create Variable

↓

Destroy Variable
```

---

Static Variable:

```c
static int count = 0;
```

Only once:

```text
Create Variable

↓

Keep Variable

↓

Reuse Variable
```

---

# Where Is It Stored?

Normal Local Variable:

```text
Stack
```

---

Static Variable:

```text
Data Segment
```

(or BSS segment if initialized to zero)

---

# Memory Layout

```text
+--------------------+
| Code Segment       |
+--------------------+
| Global Variables   |
| Static Variables   |
+--------------------+
| Heap               |
+--------------------+
| Stack              |
+--------------------+
```

Notice:

```text
Static Variable

↓

NOT on Stack
```

---

# Why Runtime Initializes Static Variables

Remember:

```text
OS

↓

Runtime

↓

main()
```

Before `main()` starts:

Runtime initializes:

```c
static int count = 0;
```

because it must already exist when the program begins.

---

# Visual Example

```c
void func()
{
    static int x = 10;
}
```

Program Starts:

```text
Runtime

↓

Create x

↓

x = 10
```

Even before:

```c
main()
```

runs.

---

# Static vs Local Variables

## Local Variable

```c
void func()
{
    int x = 10;
}
```

Memory:

```text
Function Call

↓

Create x

↓

Function Ends

↓

Destroy x
```

---

## Static Variable

```c
void func()
{
    static int x = 10;
}
```

Memory:

```text
Program Starts

↓

Create x

↓

Function Calls Reuse x

↓

Program Ends

↓

Destroy x
```

---

# Another Example

```c
void test()
{
    static int x = 5;

    printf("%d\n", x);

    x++;
}
```

Calls:

```c
test();
test();
test();
```

Output:

```text
5
6
7
```

Why?

Because:

```text
x survives
between function calls
```

---

# Why Not Use Global Variables Instead?

Example:

```c
int count = 0;
```

Global variable.

Accessible from:

```text
Every Function
```

---

Static Variable:

```c
void func()
{
    static int count = 0;
}
```

Accessible only inside:

```text
func()
```

This provides:

```text
Better Encapsulation
```

Static Variable:

Next Day Still There

# Common Mistakes

## Mistake 1

Thinking static means:

```text
Constant
```

Wrong.

Example:

```c
static int x = 10;

x++;
```

Perfectly valid.

---

## Mistake 2

Thinking static variables are on the Stack.

Wrong.

Stored in:

```text
Data Segment
```

---

## Mistake 3

Thinking static variables disappear after function ends.

Wrong.

They remain until:

```text
Program Ends
```

# Why Static Variables Exist

Without static variables:

Functions could not easily remember previous calls.

Every call would start fresh.

Static variables provide:

```text
Memory Across Function Calls
```

without using global variables.

---

# 🧠 Quick Revision

Normal Local Variable:

```c
int x;
```

* Stored on Stack
* Created on function call
* Destroyed when function ends

---

Static Variable:

```c
static int x;
```

* Stored in Data Segment
* Created once
* Keeps value forever
* Destroyed when program ends

---

Static variables were invented so functions can remember information between calls.
