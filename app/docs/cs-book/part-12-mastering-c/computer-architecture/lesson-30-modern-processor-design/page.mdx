# 1. Superscalar Execution

## Problem

Pipelining improves throughput, but the CPU still executes one instruction per clock cycle.

Engineers asked:

> What if we execute multiple instructions per cycle?

---

## Superscalar CPU

A Superscalar CPU can execute 2, 4, 6, 8+ instructions per clock cycle.

Example ‚Äî instead of:

```text
Cycle 1 ‚Üí Instruction A

Cycle 2 ‚Üí Instruction B

Cycle 3 ‚Üí Instruction C
```

CPU executes:

```text
Cycle 1

Instruction A

Instruction B

Instruction C
```

simultaneously.

---

Visual:

```text
Single Pipeline         Superscalar

Fetch                   Fetch A
  ‚Üì                     Fetch B
Decode                  Fetch C
  ‚Üì                       ‚Üì
Execute                 Decode A
                        Decode B
                        Decode C
                          ‚Üì
                        Execute A
                        Execute B
                        Execute C
```

---

## Why It Works

Many instructions are independent.

Example:

```c
a = b + c;

x = y + z;
```

No dependency. Can run simultaneously.

---

# 2. Out-of-Order Execution

## Problem

Suppose:

```c
a = b + c;

d = a + e;

x = y + z;
```

Instruction 2 waits for `a`.

But instruction 3 is independent.

A simple CPU waits ‚Äî wasting time.

---

## Out-of-Order Execution

Modern CPUs reorder instructions internally:

```text
I1 Executes

‚Üì

I3 Executes

‚Üì

I2 Executes
```

Result remains correct. Performance improves.

---

## Real-Life Analogy

Supermarket checkout ‚Äî Customer 1 has a payment issue. Instead of stopping the entire line:

```text
Serve Customer 2

Serve Customer 3

Return To Customer 1
```

Same idea.

---

# 3. Branch Prediction

## Problem

Example:

```c
if (x > 0)
{
}
```

CPU doesn't know which path will execute until the comparison finishes.

Without prediction, the pipeline stops ‚Äî very expensive.

---

## Branch Predictor

CPU guesses ‚Äî Branch Taken or Not Taken ‚Äî before the result is known.

If prediction is correct: huge speed gain.

Modern CPUs often achieve **95%‚Äì99% accuracy**.

---

# 4. Speculative Execution

Speculative Execution extends Branch Prediction.

CPU says:

```text
I Think Branch A Will Happen
```

and begins executing Branch A instructions before certainty.

---

If prediction correct: Keep Results.

If prediction wrong: Discard Results and restart.

---

Visual:

```text

‚Üì

‚Üì

Correct?

‚îú‚îÄ‚îÄ Yes ‚Üí Keep
‚îî‚îÄ‚îÄ No  ‚Üí Discard
```

# 5. Simultaneous Multithreading (SMT)

Also known as **Hyper-Threading** (Intel).

## Problem

Even modern cores sometimes sit idle ‚Äî for example, when an instruction is waiting for memory, CPU resources go unused.

---

## SMT Solution

Allow two threads inside one core.

```text
Core

‚îú‚îÄ‚îÄ Thread A
‚îî‚îÄ‚îÄ Thread B
```

When Thread A waits, Thread B uses the idle resources.

---

Example:

```text
Physical Cores:  8

SMT Threads:    16  (visible to OS)
```

# 7. SIMD Instructions

SIMD = **Single Instruction, Multiple Data**

Instead of 4 separate instructions:

```text
1+1

2+2

3+3

4+4
```

CPU performs 1 instruction with 4 results simultaneously.

---

Used heavily in:

```text
AI

Video Processing

Graphics

Scientific Computing
```

Modern examples: AVX (256-bit), AVX-512 (512-bit) ‚Äî one instruction processes many numbers.
