# Mastering Programming Language C

## Lesson 0: Roadmap — Beginner to Expert

> **Goal:** Master C 100% through building real projects — not just reading theory.

This lesson is your map. Every phase, every project, and every advanced topic you will encounter in this track is listed here. Bookmark it. Return to it often.

---

## How This Track Works

You are learning like a systems programmer, not like a student memorizing syntax.

Every lesson follows this structure:

| Step | What happens |
|------|-------------|
| 1 | Theory — the concept explained from first principles |
| 2 | Visual explanation — diagrams and analogies |
| 3 | Memory diagram — what is actually happening in RAM |
| 4 | Example — working code with every line explained |
| 5 | Exercise — you implement it |
| 6 | Challenge exercise — harder variation |
| 7 | Common mistakes — what trips people up |
| 8 | Interview questions — how this topic is tested |
| 9 | Real-world applications — where this shows up in production |

---

## Final Goal

By the end of this track you will:

- Understand C and memory management at an expert level
- Be able to read and contribute to large C codebases
- Build data structures and algorithms from scratch
- Understand operating systems, compilers, and file systems
- Write systems-level programs — shells, servers, allocators
- Debug with gdb, valgrind, and structured methodology
- Build substantial projects independently

### Phase 15 — Memory Layout

Understand every segment of a running C program:

```
High address
┌──────────────────┐
│     Stack        │  local variables, function calls
├──────────────────┤
│       ↓          │
│       ↑          │
├──────────────────┤
│     Heap         │  malloc / free
├──────────────────┤
│     BSS          │  uninitialised global variables
├──────────────────┤
│     Data         │  initialised global variables
├──────────────────┤
│     Text         │  compiled machine code
└──────────────────┘
Low address
```

## Advanced Topics — Beyond C

These topics are taught when they connect naturally to the current lesson.

### Computer Fundamentals
CPU organization, memory hierarchy, cache, registers, instruction execution, virtual memory.

### Assembly Language
x86/x86-64 basics, registers, stack operations, calling conventions, interfacing Assembly with C.
Every lesson shows what C compiles down to.

### Operating System Internals
Processes, threads, scheduler, context switching, memory management, paging, virtual memory, file systems, interrupts, system calls.

### Linux System Programming
`fork()`, `exec()`, `wait()`, `pipe()`, `dup()`, `signal()`, `mmap()`, `open()`, `read()`, `write()`, `close()` — with internal explanation for each.

### Processes and Threads
Creation, parent-child relationships, zombie and orphan processes, POSIX threads, mutexes, semaphores, condition variables, deadlocks, race conditions, atomic operations.

### Networking
TCP, UDP, IP, client-server architecture, socket programming, HTTP basics — all implemented in C.

### Embedded Programming
Memory-mapped I/O, microcontrollers, bare-metal programming, interrupts, timers, GPIO.

### Device Drivers
Kernel vs user space, driver architecture, hardware interaction, driver lifecycle.

### Compiler Design
Lexical analysis, parsing, AST generation, semantic analysis, optimisation, code generation.

### Memory Allocator
Build a simplified `malloc()` / `free()` — heap management, fragmentation, free lists, allocation strategies.

### Shell Implementation
Build a shell from scratch with pipes, redirection, and background processes.

### Database Engine
Storage engine, indexing, B-trees, query execution, transactions, logging, buffer management.

### Operating System Development
Boot process, bootloader, kernel basics, scheduling, interrupt handling, file systems — build a tiny educational OS.

### Advanced Concurrency
Lock-free programming, compare-and-swap (CAS), atomic instructions, memory ordering, concurrent data structures.

## Coding Rules

- Explain every line of code you write.
- Use meaningful variable names — no `x`, `tmp`, `foo`.
- Keep functions small and focused.
- Prefer readability over cleverness.
- Comments only where the *why* is not obvious from the code.
- Never use code you do not understand.

---

## Review Process

After every exercise:

1. **Correctness** — does it produce the right output?
2. **Efficiency** — is the time and space complexity acceptable?
3. **Readability** — would another programmer understand it?
4. **Memory safety** — are there leaks, dangling pointers, or buffer overflows?
5. **Edge cases** — does it handle empty input, zero, negatives, overflow?

---

> **Simple takeaway:** C is not just a language — it is a direct window into how computers actually work. Every concept you learn here will make you a sharper thinker about every other technology you use.
