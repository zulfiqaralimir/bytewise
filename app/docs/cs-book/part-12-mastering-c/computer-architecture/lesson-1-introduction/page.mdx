# Computer Architecture — From Zero to Expert

## Lesson 1: Introduction

> **Computer Architecture is the design and organization of a computer system, including its CPU, memory, storage, input/output devices, and the way they interact to execute instructions efficiently.**

---

## Five Main Components

### 1. Input Unit

Takes information from the outside world.

Examples: keyboard, mouse, scanner, camera, microphone.

```
You press: A
→ The keyboard sends an electrical signal to the computer.
```

---

### 2. CPU (Central Processing Unit)

The CPU is the brain. It performs arithmetic, logic, decision-making, and control.

```
5 + 3  →  CPU computes  →  8
```

---

### 3. Memory (RAM)

RAM stores data **temporarily** while a program runs.

```c
int x = 10;
```

The value `10` is placed into RAM. When power is off, it is gone.

- Very fast
- Temporary
- Volatile (loses data on shutdown)

---

### 4. Storage

Stores information **permanently**.

Examples: SSD, HDD, USB Drive.

```
photo.jpg  →  remains on disk even after shutdown
```

---

### 5. Output Unit

Shows the result of computation.

Examples: monitor, printer, speaker.

```c
printf("Hello");
// Monitor displays: Hello
```

---

## Inside the CPU

```
           CPU
     ┌──────────────┐
     │     ALU      │
     │──────────────│
     │ Control Unit │
     │──────────────│
     │  Registers   │
     └──────────────┘
```

### ALU — Arithmetic Logic Unit

Handles all computation:

```
5 + 7       → addition
10 × 8      → multiplication
15 > 9      → comparison
```

Operations: addition, subtraction, multiplication, division, comparisons, logical operations (AND / OR / NOT).

---

### Control Unit

Acts like a manager. Responsibilities:

1. **Fetch** — get the next instruction from memory
2. **Decode** — figure out what the instruction means
3. **Execute** — carry it out
4. **Coordinate** — tell every other component what to do and when

---

### Registers

Registers are tiny storage locations **inside** the CPU — the fastest memory in the computer.

Think of them as the CPU's pockets.

```
A = 5
B = 7
ALU: A + B = 12
```

The values `5` and `7` sit in registers while the ALU adds them.

---

## Memory Hierarchy

Speed decreases and capacity generally increases as you go down:

```
Fast  ↑
      │  Registers
      │  Cache
      │  RAM
      │  SSD
      │  HDD
      │  Tape
Slow  ↓
```

---

## The Von Neumann Architecture

Most modern computers follow this design.

```
           Memory
              ▲
              │
Input → CPU → Output
              │
              ▼
           Storage
```

**Key idea:** Instructions and data are stored in the **same** memory. The CPU reads both from RAM.

---

## How a Program Runs — Step by Step

```c
int a = 5;
int b = 3;
int c = a + b;
```

| Step | What happens |
|------|-------------|
| 1 | Program sits on SSD |
| 2 | OS loads it into RAM |
| 3 | CPU fetches the instruction |
| 4 | CPU decodes the instruction |
| 5 | ALU performs the addition |
| 6 | Result stored in memory |
| 7 | Output displayed |

---

## The Fetch–Decode–Execute Cycle

Every CPU continuously repeats this loop — billions of times per second:

```
Fetch → Decode → Execute → Fetch → Decode → Execute → ...
```

---

## Where C Fits In

When you write:

```c
int x = 5;
```

the **compiler** translates it into machine instructions the CPU actually understands:

```
LOAD
STORE
ADD
MOV
```

The CPU only understands **machine code** — C is an abstraction on top of it.

## Key Definitions to Remember

**CPU** — the brain; fetches, decodes, and executes instructions.

**ALU** — the part of the CPU that does math and logic.

**Control Unit** — the part of the CPU that coordinates everything.

**Registers** — the fastest, smallest storage; lives inside the CPU.

**RAM** — fast temporary memory; holds running programs.

**Storage** — slow permanent memory; holds files between sessions.

**Von Neumann Architecture** — the design where instructions and data share the same memory.

**Fetch–Decode–Execute** — the cycle the CPU repeats endlessly.

**Compiler** — the tool that translates C into machine code the CPU can run.
