# Topic 1: Arrays & Hashing

---

## WHY тАФ Why Does This Topic Exist?

Imagine you have a list of 1 million numbers and someone asks: *"Does the number 742 appear twice?"*

The naive approach checks every pair тАФ that is **1 trillion comparisons**. Completely unusable.

Arrays & Hashing solves this by trading **space for speed**. You store what you have seen in a structure that answers *"have I seen this?"* in **O(1)** тАФ instantly, regardless of input size.

This is the most fundamental trade-off in all of computer science: **time vs space**.

---

## WHAT тАФ The Data Structures

**Array** тАФ ordered, indexed, O(1) access by index, O(n) search.

**HashMap** (`dict` in Python) тАФ stores keyтЖТvalue pairs. O(1) average for insert, lookup, and delete.

**HashSet** (`set` in Python) тАФ stores unique keys only. O(1) average for insert, lookup, and delete.

Both use **hashing** internally тАФ a function that converts any key into an array index in O(1).

---

## HOW тАФ Recognizing This Pattern

When you see any of these phrases in a problem, think Arrays & Hashing:

| Trigger Phrase | Tool to Reach For |
|---|---|
| "find duplicate" / "already seen" | `set` |
| "frequency of elements" | `Counter` or `dict` |
| "two elements that sum to X" | `dict` for complement lookup |
| "group by some property" | `defaultdict(list)` |
| "count occurrences" | `Counter` |
