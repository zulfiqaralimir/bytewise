## 3. The Problem-Solving Learning Path

Every problem in this chapter followed the same structured learning path. This is the framework you should apply to every new problem you encounter:

> **Problem → Constraints → Hint → Your Attempt → Brute Force → Why It Fails → Intuition → Optimized Approach → Python Code → Complexity Analysis → Edge Cases**

### What Each Step Means

| Step | What to Do |
|------|-----------|
| **Problem Statement** | Read carefully. Underline key verbs: find, count, return, minimize. |
| **Constraints** | These are hints. Small n means O(n²) is fine. Large n demands O(n log n) or better. |
| **Hint** | A gentle nudge toward the pattern without giving away the solution. |
| **Your Attempt** | Always try first. Even a wrong attempt builds intuition. |
| **Brute Force** | Understand the naive O(n²) or O(2ⁿ) solution before optimizing. |
| **Why It Fails** | Time Limit Exceeded (TLE) or Memory Limit Exceeded (MLE) reasoning. |
| **Intuition** | The core insight that unlocks the optimized approach. |
| **Optimized Approach** | The clean, efficient algorithm with step-by-step reasoning. |
| **Python Code** | Interview-quality implementation with meaningful variable names. |
| **Complexity Analysis** | Time and Space complexity with justification. |
| **Edge Cases** | Empty input, single element, all duplicates, negative numbers, overflow. |

### Key Python Patterns Worth Memorizing

```python
# Frequency counter
from collections import Counter
freq = Counter(nums)              # {element: count}

# Default dictionary
from collections import defaultdict
graph = defaultdict(list)         # No KeyError on missing key

# Min-heap (Python default)
import heapq
heapq.heappush(heap, val)         # Push
heapq.heappop(heap)               # Pop smallest

# Max-heap (negate values)
heapq.heappush(heap, -val)        # Negate to simulate max-heap

# Memoization
from functools import lru_cache
@lru_cache(maxsize=None)          # Cache recursive results
def solve(i): ...

# Binary search
import bisect
bisect.bisect_left(arr, target)   # Leftmost insertion index
```

## 6. Complexity Quick Reference

| Topic | Time | Space | Notes |
|-------|------|-------|-------|
| Arrays & Hashing | O(n) | O(n) | HashMap/HashSet storage |
| Two Pointers | O(n log n) | O(1) | Sort first, then O(n) scan |
| Sliding Window | O(n) | O(k) | Window contents only |
| Stack | O(n) | O(n) | Stack grows to input size |
| Binary Search | O(log n) | O(1) | No extra space needed |
| Linked List | O(n) | O(1) | In-place pointer manipulation |
| Trees | O(n) | O(h) | h = tree height, O(log n) balanced |
| Heap | O(n log k) | O(k) | k = heap size |
| Backtracking | O(2ⁿ) / O(n!) | O(n) | Exponential — prune early |
| Tries | O(m) | O(m·n) | m = word length, n = words |
| Graphs | O(V + E) | O(V) | Visited set size |
| Advanced Graphs | O(E log V) | O(V + E) | Dijkstra with priority queue |
| 1-D DP | O(n) | O(n) → O(1) | Often space-optimizable |
| 2-D DP | O(m × n) | O(m × n) | Grid DP table |
| Greedy | O(n log n) | O(1) | Dominated by sort |
| Intervals | O(n log n) | O(n) | Output intervals storage |
| Math & Geometry | O(√n) / O(log n) | O(1) | Depends on algorithm |
| Bit Manipulation | O(log n) / O(1) | O(1) | Bit-level ops are O(1) |

## 7. Common Mistakes & How to Avoid Them

| Topic | Most Common Mistake |
|-------|-------------------|
| Arrays & Hashing | Using list instead of set → O(n) lookup becomes O(n²) |
| Two Pointers | Not sorting first when required; moving both pointers simultaneously |
| Sliding Window | Forgetting to shrink window; not tracking window max correctly |
| Stack | Not handling empty stack before pop; forgetting unmatched brackets edge case |
| Binary Search | Off-by-one in lo/hi boundaries; infinite loop when mid not excluded |
| Linked List | Losing reference before reassigning next; not handling head=None |
| Trees | Confusing in-order / pre-order / post-order; not handling null children |
| Heap | Using max-heap in Python (negate values); pushing wrong tuple order |
| Backtracking | Not copying path on append (shallow copy bug); not undoing state on backtrack |
| Tries | Confusing end-of-word flag with empty children; not initializing children dict |
| Graphs | Not marking visited before recursion (infinite loop); forgetting disconnected components |
| Advanced Graphs | Assuming Dijkstra works with negative weights; wrong relaxation order |
| 1-D DP | Accessing dp[i-1] when i=0; confusing tabulation with memoization |
| 2-D DP | Wrong base case initialization; off-by-one in row/col indices |
| Greedy | Assuming greedy always works — must prove local optimum = global optimum |
| Intervals | Not sorting by start time; missing the merge condition boundary |
| Math & Geometry | Integer overflow in Python is rare but watch multiplication in constraints |
| Bit Manipulation | Operator precedence: always use parentheses around bitwise expressions |

## 8. Interview Readiness Checklist

Before walking into a coding interview, go through this checklist:

- ☐ I can identify the correct pattern within 2 minutes of reading a problem
- ☐ I always clarify constraints before coding (array size, negative numbers, duplicates)
- ☐ I think through edge cases before writing any code
- ☐ I can explain brute force and why it fails before jumping to the optimized solution
- ☐ I code cleanly with meaningful variable names (not i, j, k for everything)
- ☐ I analyze time and space complexity for every solution I write
- ☐ I test my solution with the given examples, then with my own edge cases
- ☐ I know the key Python built-ins: Counter, defaultdict, heapq, deque, bisect
- ☐ I can implement BFS and DFS from memory on a graph and a tree
- ☐ I understand when to use iterative vs recursive solutions
- ☐ I can explain memoization and convert a recursive solution to tabulation
- ☐ I have practiced at least 3 problems from every topic category
- ☐ I can code a binary search without looking it up — including the boundary conditions
- ☐ I know how to detect a cycle in a linked list (Floyd's algorithm)
- ☐ I understand the difference between Dijkstra, Bellman-Ford, and Floyd-Warshall
