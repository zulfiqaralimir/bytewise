# Topic 1: Problem 2 of 9

## Valid Anagram

**LeetCode #242 · 🟢 Easy**

---

### Problem Statement

> Given two strings `s` and `t`, return `true` if `t` is an anagram of `s`, and `false` otherwise.

> An **anagram** is a word formed by rearranging the letters of another word, using all original letters exactly once.

**Examples:**

```
Input:  s = "anagram", t = "nagaram"  →  Output: true
Input:  s = "rat",     t = "car"      →  Output: false
```

**Constraints:**
- `1 <= s.length, t.length <= 5 * 10⁴`
- `s` and `t` consist of lowercase English letters only

---

### Brute Force Approach

Sort both strings and compare:

```python
def isAnagram(s, t):
    return sorted(s) == sorted(t)
```

**Time Complexity: O(n log n)** — sorting costs O(n log n).

**Space Complexity: O(n)** — `sorted()` creates new lists.

**Why it fails:** This actually passes LeetCode, but interviewers will ask: *"Can you do it in O(n)?"* Sorting is unnecessary when we only care about character frequencies.

---

### Intuition

Two strings are anagrams if and only if **every character appears the same number of times in both**.

Count the frequency of each character in `s`, count the frequency in `t`, and compare. A `Counter` or `dict` does this in one pass each — O(n) total.

---

### Optimized Approach

1. If lengths differ → immediately return `False`
2. Build a frequency map for `s` using `Counter`
3. Build a frequency map for `t` using `Counter`
4. If both maps are equal → return `True`, else `False`

---

### Python Solution

```python
from collections import Counter

def isAnagram(s, t):
    if len(s) != len(t):
        return False
    return Counter(s) == Counter(t)
```

**Manual dict approach (without Counter):**

```python
def isAnagram(s, t):
    if len(s) != len(t):
        return False

    count = {}
    for c in s:
        count[c] = count.get(c, 0) + 1
    for c in t:
        count[c] = count.get(c, 0) - 1
        if count[c] < 0:
            return False
    return True
```

> The manual approach increments counts for `s` and decrements for `t`. If any count goes negative, `t` has a character that `s` does not — not an anagram.

---


### 📌 Deep Dive — Understanding the Manual Dict Approach

The idea is simple: use one dictionary to track both strings at once.

- For every character in `s` → **add 1**
- For every character in `t` → **subtract 1**
- At the end, if all counts are **zero** → same frequency → anagram ✅
- If any count goes **negative** → `t` has something `s` doesn't → not anagram ❌

---

#### Step by Step with Example

s = "anagram"

t = "nagaram"

**Phase 1 — Loop through `s`, increment counts:**

```python
for c in s:
    count[c] = count.get(c, 0) + 1
```

| Character | count.get(c, 0) | After +1 | count dict |
|---|---|---|---|
| `a` | 0 (not seen) | 1 | `{'a': 1}` |
| `n` | 0 | 1 | `{'a': 1, 'n': 1}` |
| `a` | 1 (seen before) | 2 | `{'a': 2, 'n': 1}` |
| `g` | 0 | 1 | `{'a': 2, 'n': 1, 'g': 1}` |
| `r` | 0 | 1 | `{'a': 2, 'n': 1, 'g': 1, 'r': 1}` |
| `a` | 2 | 3 | `{'a': 3, 'n': 1, 'g': 1, 'r': 1}` |
| `m` | 0 | 1 | `{'a': 3, 'n': 1, 'g': 1, 'r': 1, 'm': 1}` |

**Phase 2 — Loop through `t`, decrement counts:**

```python
for c in t:
    count[c] = count.get(c, 0) - 1
    if count[c] < 0:
        return False
```

| Character | count before | After -1 | < 0? |
|---|---|---|---|
| `n` | 1 | 0 | No |
| `a` | 3 | 2 | No |
| `g` | 1 | 0 | No |
| `a` | 2 | 1 | No |
| `r` | 1 | 0 | No |
| `a` | 1 | 0 | No |
| `m` | 1 | 0 | No |

All counts reached zero → `return True` ✅

---

#### What Does `count.get(c, 0)` Mean?

```python
count = {}

# Normal dict access — crashes if key missing
count['a']          # KeyError! 'a' not in dict yet

# .get(key, default) — safe, returns default if missing
count.get('a', 0)   # Returns 0 instead of crashing
```

So `count.get(c, 0) + 1` means: *"Give me the current count of this character, or 0 if I have never seen it, then add 1."*

---

#### When Would Count Go Negative?

Try `s = "rat"`, `t = "car"`:

After Phase 1: `count = {'r': 1, 'a': 1, 't': 1}`

| Character | count before | After -1 | < 0? |
|---|---|---|---|
| `c` | 0 (never seen) | -1 | YES → return False ❌ |

`t` contains `c` but `s` never did → count goes negative → immediately caught.

---

#### Why Check `< 0` Inside the Loop?

Checking inside the loop is an **early exit optimization** — the moment we know it is not an anagram, we stop immediately without processing the rest of `t`.

---

#### Summary — Think of It as a Bank Account

```python
count[c] = count.get(c, 0) + 1   # s deposits characters
count[c] = count.get(c, 0) - 1   # t withdraws characters
if count[c] < 0: return False     # overdraft = not an anagram
```

- `s` deposits characters into the account
- `t` withdraws characters from the account
- If `t` tries to withdraw something never deposited → overdraft → not an anagram
---

### Complexity Analysis

| | Complexity | Reason |
|---|---|---|
| **Time** | O(n) | One pass each through `s` and `t` |
| **Space** | O(1) | At most 26 keys — fixed alphabet size |

> Space is technically O(1) because there are only 26 possible lowercase letters. The frequency map never grows beyond 26 entries regardless of input size.

---

### Edge Cases

| Input | Expected Output | Reason |
|---|---|---|
| `s="a", t="a"` | `true` | Single identical characters |
| `s="ab", t="a"` | `false` | Different lengths — early exit |
| `s="aa", t="bb"` | `false` | Same length, different characters |
| `s="aab", t="bba"` | `false` | Frequency mismatch |
| `s="", t=""` | `true` | Two empty strings are anagrams |

---

### Python Concept Spotlight — `Counter`

`Counter` from `collections` is one of the most powerful tools in competitive Python:

```python
from collections import Counter

Counter("anagram")
# Counter({'a': 3, 'n': 1, 'g': 1, 'r': 1, 'm': 1})

Counter("anagram") == Counter("nagaram")
# True — order does not matter, only frequency counts
```

---

### Key Takeaway

> **"Do these two things have the same composition?"** → build a `Counter` for each and compare.

---

### Real-World Application

Search engines — detect synonym or rearranged keyword queries to return relevant results even when word order differs.
