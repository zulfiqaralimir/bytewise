# Section 1.6 Networks Under Attack (Part 6C)

# Denial-of-Service (DoS) and Distributed Denial-of-Service (DDoS) Attacks

In the previous part, we learned about:

- Malware
- Viruses
- Worms
- Trojan Horses
- Spyware
- Ransomware
- Botnets

Now another important question arises.

> <span style={{ color: "#38bdf8", fontWeight: 700 }}>What if an attacker does not want to steal your data?</span>

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Instead,</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>the attacker wants to make your service completely unavailable.</span>

For example,

suppose attackers want to shut down:

- Amazon
- Google
- Facebook
- Netflix
- A Bank Website
- An Online Shopping Store

<span style={{ color: "#14b8a6", fontWeight: 700 }}>without physically damaging the servers.</span>

How can they do this?

The answer is:

> **Denial-of-Service (DoS) Attacks**

---

# What Is a Denial-of-Service (DoS) Attack?

Definition:

> **A Denial-of-Service (DoS) attack is an attack that attempts to make a network, computer, or online service unavailable to legitimate users** <span style={{ color: "#d946ef", fontWeight: 700 }}>by overwhelming it with malicious traffic or requests.</span>

Notice the objective.

The attacker does **not** necessarily want to:

- Read your files.
- Steal passwords.
- Modify data.

Instead,

the attacker wants to prevent legitimate users from accessing the service.

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>This violates one of the CIA Triad principles:</span>

> <span style={{ color: "#eab308", fontWeight: 700 }}>Availability</span>

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Real-Life Analogy</span>

Imagine a restaurant.

Normally,

customers enter,

order food,

and leave.

Now suppose someone hires hundreds of people.

These people enter the restaurant,

occupy every table,

but never order anything.

Real customers arrive,

but there are no seats available.

The restaurant is technically open,

yet nobody can use it.

A DoS attack works in exactly the same way.

---

# Goal of a DoS Attack

<span style={{ color: "#84cc16", fontWeight: 700 }}>The attacker's objective is to exhaust one or more resources.</span>

Examples include:

- <span style={{ color: "#f43f5e", fontWeight: 700 }}>CPU</span>
- <span style={{ color: "#fbbf24", fontWeight: 700 }}>Memory (RAM)</span>
- <span style={{ color: "#38bdf8", fontWeight: 700 }}>Network Bandwidth</span>
- <span style={{ color: "#84cc16", fontWeight: 700 }}>Disk Space</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Database Connections</span>
- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Application Threads</span>

Once these resources are exhausted,

legitimate users experience:

- Slow responses.
- Connection failures.
- Service outages.

---

# How Does a DoS Attack Work?

Suppose a web server normally receives:

```text
100 Requests/Second
```

The server can comfortably process them.

Now an attacker sends:

```text
500,000 Requests/Second
```

The server becomes overwhelmed.

Result:

```text
Legitimate Requests

↓

Delayed

↓

Rejected

↓

Dropped
```

---

# Normal Operation

```text
Users

↓

Server

↓

Normal Responses
```

Everything works correctly.

---

# During a DoS Attack

```text
Attacker

↓

Millions of Requests

↓

Server

↓

Overloaded

↓

Real Users Cannot Connect
```

---

# Why Is It Called "Denial of Service"?

Because the attacker is denying legitimate users access to the service.

The service still exists,

but users cannot use it.

---

# Types of Resources That Can Be Exhausted

A DoS attack may target different resources.

---

## CPU Exhaustion

<span style={{ color: "#f43f5e", fontWeight: 700 }}>The attacker sends requests requiring heavy computation.</span>

Eventually,

the CPU becomes fully occupied.

---

## Memory Exhaustion

<span style={{ color: "#fbbf24", fontWeight: 700 }}>The attacker forces the server to allocate memory repeatedly.</span>

Eventually,

RAM becomes unavailable.

---

## Bandwidth Exhaustion

<span style={{ color: "#38bdf8", fontWeight: 700 }}>The attacker floods the communication link with enormous traffic.</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>Even before packets reach the server,</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>the network connection becomes saturated.</span>

---

## Application Exhaustion

<span style={{ color: "#14b8a6", fontWeight: 700 }}>The attacker repeatedly requests expensive operations.</span>

Examples:

- Database queries.
- File generation.
- Search operations.

The application becomes overloaded.

---

# Flooding Attacks

Many DoS attacks are called:

> **Flooding Attacks**

Definition:

> **A Flooding Attack overwhelms a target** <span style={{ color: "#f43f5e", fontWeight: 700 }}>by sending an extremely large volume of traffic.</span>

Think of a flood.

Water continues entering until everything becomes overwhelmed.

Networking uses the same idea.

---

# Example

Suppose:

Server Capacity:

```text
1 Gbps
```

Attacker sends:

```text
5 Gbps
```

The server cannot process all incoming traffic.

Users experience delays or complete failure.

---

# Distributed Denial-of-Service (DDoS)

Now let's study an even more dangerous attack.

---

# What Is a DDoS Attack?

Definition:

> **A Distributed Denial-of-Service (DDoS) attack** <span style={{ color: "#38bdf8", fontWeight: 700 }}>uses many compromised computers to simultaneously attack a single target.</span>

Notice the word:

> **Distributed**

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Instead of one attacker,</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>thousands or even millions of computers participate.</span>

---

# Why Is DDoS More Dangerous?

Imagine one person trying to block a highway.

Difficult.

Now imagine:

```text
100,000 Vehicles
```

blocking every road simultaneously.

Much more effective.

DDoS works exactly the same way.

---

# Where Do Attackers Get Thousands of Computers?

Remember what we studied earlier.

<span style={{ color: "#38bdf8", fontWeight: 700 }}>Attackers use:</span>

> <span style={{ color: "#fbbf24", fontWeight: 700 }}>Botnets</span>

---

# Botnet Review

A Botnet consists of:

```text
Bot

+

Bot

+

Bot

+

Bot

+

Thousands More
```

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Each infected computer waits for commands.</span>

<span style={{ color: "#14b8a6", fontWeight: 700 }}>When instructed,</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>every bot attacks the same target simultaneously.</span>

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>DDoS Architecture</span>

```text
                 Attacker

                     │

             Command Server

     ┌─────────┼─────────┐

     ▼         ▼         ▼

   Bot 1     Bot 2     Bot 3

     ▼         ▼         ▼

   Victim Server
```

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Thousands of bots send traffic simultaneously.</span>

---

# Example

Suppose:

Each bot sends:

```text
2 Mbps
```

Now imagine:

```text
500,000 Bots
```

Total attack traffic:

```text
2 Mbps

×

500,000

=

1,000,000 Mbps

=

1 Tbps
```

<span style={{ color: "#eab308", fontWeight: 700 }}>Even large Internet services struggle against attacks of this magnitude.</span>

---

# Why Is DDoS Difficult to Stop?

Because traffic comes from:

- Thousands of countries.
- Millions of IP addresses.
- <span style={{ color: "#84cc16", fontWeight: 700 }}>Legitimate-looking computers.</span>

Blocking one IP address has little effect.

---

# Reflection Attacks

Attackers sometimes use another technique called:

> **Reflection**

---

# What Is Reflection?

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Instead of attacking directly,</span>

<span style={{ color: "#38bdf8", fontWeight: 700 }}>the attacker tricks another server into sending traffic to the victim.</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>The third-party server becomes an unwilling participant.</span>

---

# Real-Life Analogy

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Imagine sending someone else's address to thousands of stores,</span>

<span style={{ color: "#14b8a6", fontWeight: 700 }}>asking them to mail free catalogs.</span>

The stores send catalogs to the innocent victim.

The stores are not malicious.

They were simply tricked.

Reflection attacks work similarly.

---

# Reflection Attack Diagram

```text
Attacker

↓

Fake Request

↓

Public Server

↓

Large Response

↓

Victim
```

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>The public server unknowingly attacks the victim.</span>

---

# IP Spoofing and Reflection

<span style={{ color: "#eab308", fontWeight: 700 }}>Reflection attacks often rely on:</span>

> <span style={{ color: "#84cc16", fontWeight: 700 }}>IP Spoofing</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>The attacker forges the source IP address,</span>

<span style={{ color: "#38bdf8", fontWeight: 700 }}>making it appear that the victim sent the request.</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>The public server replies to the victim instead of the attacker.</span>

We will study IP Spoofing in detail later.

---

# Amplification Attacks

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Reflection becomes even more dangerous when combined with:</span>

> <span style={{ color: "#14b8a6", fontWeight: 700 }}>Amplification</span>

---

# What Is Amplification?

Definition:

> **An Amplification Attack generates a response much larger than the original request.**

---

# Example

Suppose the attacker sends:

```text
100 Bytes
```

The server responds with:

```text
5,000 Bytes
```

Amplification Factor:

```text
5000

/

100

=

50
```

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Every request becomes fifty times larger.</span>

<span style={{ color: "#eab308", fontWeight: 700 }}>Thousands of servers can generate enormous traffic.</span>

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Common Amplification Services</span>

Historically,

attackers have abused services such as:

- DNS
- NTP
- Memcached

These services can produce responses much larger than the original requests when improperly configured.

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Reflection + Amplification</span>

```text
Attacker

↓

Small Fake Request

↓

Public Server

↓

Large Response

↓

Victim
```

<span style={{ color: "#f43f5e", fontWeight: 700 }}>The attacker sends very little traffic,</span>

<span style={{ color: "#38bdf8", fontWeight: 700 }}>yet the victim receives massive amounts.</span>

---

# Real-World Example

Imagine shouting into a cave.

A small voice produces a large echo.

Amplification attacks behave similarly.

Small requests generate much larger responses.

---

# Effects of DoS and DDoS Attacks

Victims may experience:

- Slow websites.
- Failed logins.
- Dropped connections.
- Interrupted video streaming.
- Online gaming failures.
- Banking service outages.

Businesses may lose:

- Revenue.
- Customers.
- Reputation.

---

# Famous DDoS Attack

<span style={{ color: "#d946ef", fontWeight: 700 }}>One of the most well-known attacks occurred in 2016.</span>

The **Mirai Botnet** infected hundreds of thousands of <span style={{ color: "#38bdf8", fontWeight: 700 }}>Internet of Things (IoT)</span> devices such as:

- Security Cameras
- <span style={{ color: "#f43f5e", fontWeight: 700 }}>Routers</span>
- <span style={{ color: "#fbbf24", fontWeight: 700 }}>DVRs</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>These compromised devices launched a massive DDoS attack against DNS provider Dyn.</span>

As a result,

many popular websites became temporarily unavailable, including services from companies such as <span style={{ color: "#14b8a6", fontWeight: 700 }}>Twitter, Netflix, Reddit, GitHub, and Spotify.</span>

---

# How Can We Defend Against DoS and DDoS?

<span style={{ color: "#a5b4fc", fontWeight: 700 }}>There is no single solution,</span>

<span style={{ color: "#a5b4fc", fontWeight: 700 }}>but several techniques reduce risk.</span>

Examples include:

- <span style={{ color: "#f43f5e", fontWeight: 700 }}>Firewalls.</span>
- <span style={{ color: "#fbbf24", fontWeight: 700 }}>Rate Limiting.</span>
- <span style={{ color: "#38bdf8", fontWeight: 700 }}>Traffic Filtering.</span>
- <span style={{ color: "#84cc16", fontWeight: 700 }}>Load Balancing.</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Content Delivery Networks (CDNs).</span>
- DDoS Mitigation Services.
- Anycast Routing.
- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Intrusion Detection Systems.</span>

We will study several of these mechanisms in the next section.

---

# Rate Limiting

One simple defense is:

> **Rate Limiting**

Definition:

> Restricting how many requests a client may send during a given time period.

Example:

```text
Maximum:

100 Requests/Minute
```

Additional requests are delayed or rejected.

---

# Load Balancing

Instead of one server,

traffic is distributed among multiple servers.

```text
Users

↓

Load Balancer

↓

Server 1

Server 2

Server 3
```

This prevents one server from becoming overwhelmed.

---

# Content Delivery Networks (CDNs)

A CDN stores content on servers distributed around the world.

Benefits include:

- Reduced latency.
- Improved availability.
- Better resistance against DDoS attacks by distributing traffic.

---

# <span style={{ color: "#0ea5e9", fontWeight: 700 }}>Anycast Routing</span>

With Anycast,

<span style={{ color: "#38bdf8", fontWeight: 700 }}>multiple servers share the same IP address.</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>Traffic automatically reaches the nearest or best-performing server.</span>

During an attack,

<span style={{ color: "#d946ef", fontWeight: 700 }}>traffic is spread across many locations instead of overwhelming a single data center.</span>

---

# DoS vs DDoS

| DoS | DDoS |
|------|------|
| Single attacking source | Many attacking sources |
| Easier to identify | Much harder to identify |
| Smaller attack scale | Much larger attack scale |
| Simpler to mitigate | More difficult to mitigate |
| Usually one compromised system | Usually thousands or millions of bots |

---

# Common Misconceptions

## Misconception 1

> A DoS attack steals confidential information.

Incorrect.

<span style={{ color: "#f43f5e", fontWeight: 700 }}>A DoS attack primarily targets **Availability**, not Confidentiality.</span>

---

## Misconception 2

> <span style={{ color: "#fbbf24", fontWeight: 700 }}>Every DDoS attack comes from hackers using their own computers.</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Incorrect.</span>

<span style={{ color: "#14b8a6", fontWeight: 700 }}>Most DDoS attacks originate from compromised devices within Botnets.</span>

---

## Misconception 3

> Bigger servers cannot be attacked.

Incorrect.

Even very large organizations can experience DDoS attacks if the attack volume exceeds available resources.

---

## Misconception 4

> Blocking one IP address stops a DDoS attack.

Incorrect.

A DDoS attack may involve hundreds of thousands of IP addresses.

---

# Key Takeaways

- A Denial-of-Service (DoS) attack attempts to make a service unavailable by exhausting its resources.
- DoS attacks primarily target the **Availability** component of the CIA Triad.
- A Distributed Denial-of-Service (DDoS) attack uses many compromised devices simultaneously.
- Botnets are commonly used to launch DDoS attacks.
- Reflection attacks trick third-party servers into sending traffic to the victim.
- Amplification attacks generate responses much larger than the original requests.
- Modern defenses include rate limiting, load balancing, CDNs, Anycast routing, and DDoS mitigation services.

---

# Technical Vocabulary

| Term | Definition |
|------|------------|
| Denial-of-Service (DoS) Attack | Attack that attempts to make a service unavailable by exhausting system or network resources. |
| Distributed Denial-of-Service (DDoS) Attack | DoS attack launched simultaneously from many compromised computers. |
| Flooding Attack | Attack that overwhelms a target by sending excessive traffic or requests. |
| Botnet | Network of compromised computers controlled by an attacker. |
| Reflection Attack | Attack in which third-party servers are tricked into sending traffic to the victim. |
| Amplification Attack | Attack that generates responses much larger than the original requests, increasing attack traffic. |
| Rate Limiting | Security mechanism that restricts the number of requests a client can make in a given time. |
| Load Balancer | Device or software that distributes traffic across multiple servers. |
| Content Delivery Network (CDN) | Distributed network of servers that improves content delivery and helps absorb traffic surges. |
| Anycast Routing | Routing technique in which multiple geographically distributed servers share the same IP address to distribute incoming traffic. |
