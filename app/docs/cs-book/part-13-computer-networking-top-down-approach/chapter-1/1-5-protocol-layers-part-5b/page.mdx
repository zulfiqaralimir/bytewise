# Section 1.5 Protocol Layers and Their Service Models (Part 5B)

# <span style={{ color: "#38bdf8", fontWeight: 700 }}>The Internet Protocol Stack (Five-Layer Model)</span>

In the previous part, we learned **why networking is divided into layers**.

Now another important question arises.

> **How many layers does the Internet actually use?**

There are <span style={{ color: "#fbbf24", fontWeight: 700 }}>two famous networking models:</span>

- The <span style={{ color: "#f43f5e", fontWeight: 700 }}>Internet Protocol Stack (5-layer model) (used in Kurose)</span>
- The <span style={{ color: "#14b8a6", fontWeight: 700 }}>OSI Reference Model (7-layer model)</span>

In this chapter, Kurose first explains the **Internet Protocol Stack**, because <span style={{ color: "#eab308", fontWeight: 700 }}>it represents how the Internet actually works.</span>

Later, we will compare it with the OSI model.

---

# What Is a Protocol Stack?

Definition:

> **A Protocol Stack is a collection of protocol layers that work together to provide end-to-end communication between networked devices.**

<span style={{ color: "#d946ef", fontWeight: 700 }}>Each layer:</span>

- <span style={{ color: "#0ea5e9", fontWeight: 700 }}>Performs a specific task.</span>
- <span style={{ color: "#84cc16", fontWeight: 700 }}>Uses services from the layer below.</span>
- <span style={{ color: "#fb7185", fontWeight: 700 }}>Provides services to the layer above.</span>

<span style={{ color: "#c084fc", fontWeight: 700 }}>Together,</span>

<span style={{ color: "#60a5fa", fontWeight: 700 }}>all layers enable complete communication.</span>

---

# The Internet Protocol Stack

The Internet Protocol Stack consists of **five layers**.

This is one of the most important diagrams in computer networking.

You should <span style={{ color: "#86efac", fontWeight: 700 }}>memorize it.</span>

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Understanding the Layer Order</span>

Notice the order carefully.

```text
Application

↓

Transport

↓

Network

↓

Link

↓

Physical
```

<span style={{ color: "#38bdf8", fontWeight: 700 }}>When sending data,</span>

communication moves <span style={{ color: "#fbbf24", fontWeight: 700 }}>from top to bottom.</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>When receiving data,</span>

communication moves <span style={{ color: "#14b8a6", fontWeight: 700 }}>from bottom to top.</span>

---

# Why This Order?

<span style={{ color: "#eab308", fontWeight: 700 }}>Each layer depends on the services provided by the layer below it.</span>

For example,

<span style={{ color: "#d946ef", fontWeight: 700 }}>The Application Layer cannot directly send electrical signals.</span>

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Instead,</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>it requests help from:</span>

```text
Transport Layer

↓

Network Layer

↓

Link Layer

↓

Physical Layer
```

Finally,

the Physical Layer sends the actual bits.

---

# Overview of Each Layer

Before studying each layer individually,

let's briefly understand their responsibilities.

| <span style={{ color: "#fb7185", fontWeight: 700 }}>Layer</span> | <span style={{ color: "#c084fc", fontWeight: 700 }}>Main Responsibility</span> |
|--------|----------------------|
| <span style={{ color: "#60a5fa", fontWeight: 700 }}>Application</span> | <span style={{ color: "#60a5fa", fontWeight: 700 }}>Provides network services to applications.</span> |
| <span style={{ color: "#e879f9", fontWeight: 700 }}>Transport</span> | <span style={{ color: "#e879f9", fontWeight: 700 }}>Provides end-to-end communication between processes.</span> |
| <span style={{ color: "#fde047", fontWeight: 700 }}>Network</span> | <span style={{ color: "#fde047", fontWeight: 700 }}>Determines how packets travel across multiple networks.</span> |
| <span style={{ color: "#fca5a5", fontWeight: 700 }}>Link</span> | <span style={{ color: "#fca5a5", fontWeight: 700 }}>Transfers frames between directly connected devices.</span> |
| <span style={{ color: "#93c5fd", fontWeight: 700 }}>Physical</span> | <span style={{ color: "#93c5fd", fontWeight: 700 }}>Transmits raw bits through the communication medium.</span> |

Think of each layer as answering a different question.

---

# Application Layer

## Main Responsibility

The Application Layer provides network services directly to user applications.

Notice something important.

<span style={{ color: "#38bdf8", fontWeight: 700 }}>The Application Layer is not the application itself.</span>

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Instead,</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>it provides networking support to applications.</span>

---

# Examples

Applications include:

- Web Browser
- Email Client
- WhatsApp
- Zoom
- Google Chrome
- Firefox
- Microsoft Outlook

These applications use Application Layer protocols to communicate.

---

# <span style={{ color: "#14b8a6", fontWeight: 700 }}>Common Application Layer Protocols</span>

Examples include:

- <span style={{ color: "#eab308", fontWeight: 700 }}>HTTP</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>HTTPS</span>
- <span style={{ color: "#0ea5e9", fontWeight: 700 }}>FTP</span>
- <span style={{ color: "#84cc16", fontWeight: 700 }}>SMTP</span>
- <span style={{ color: "#fb7185", fontWeight: 700 }}>POP3</span>
- <span style={{ color: "#c084fc", fontWeight: 700 }}>IMAP</span>
- <span style={{ color: "#60a5fa", fontWeight: 700 }}>DNS</span>
- <span style={{ color: "#e879f9", fontWeight: 700 }}>DHCP</span>
- <span style={{ color: "#fde047", fontWeight: 700 }}>SSH</span>

Each protocol serves a different purpose.

---

# Example

Suppose you open:

```text
https://www.google.com
```

The browser uses:

```text
HTTP

or

HTTPS
```

to request a webpage.

<span style={{ color: "#fca5a5", fontWeight: 700 }}>The Application Layer creates the message.</span>

---

# Application Layer Data

<span style={{ color: "#93c5fd", fontWeight: 700 }}>At this layer,</span>

<span style={{ color: "#fdba74", fontWeight: 700 }}>the information is simply called:</span>

```text
Message
```

Example:

```text
GET /index.html HTTP/1.1
```

The Application Layer does not worry about:

- Routers
- IP Addresses
- Ethernet
- WiFi

<span style={{ color: "#86efac", fontWeight: 700 }}>Its job is simply to create application data.</span>

---

# Transport Layer

<span style={{ color: "#a5b4fc", fontWeight: 700 }}>The Application Layer now passes its message downward.</span>

The next layer is:

> **Transport Layer**

---

# Main Responsibility

The Transport Layer provides:

> Logical communication between <span style={{ color: "#38bdf8", fontWeight: 700 }}>application processes</span> running on different hosts.

Notice the words:

> **Processes**

Not computers.

This is an important distinction.

---

# Real-Life Analogy

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Imagine a hotel.</span>

The building represents:

```text
Computer
```

Each room represents:

```text
Application Process
```

A courier must deliver a parcel to:

Not just the hotel,

but to the correct room.

Similarly,

<span style={{ color: "#f43f5e", fontWeight: 700 }}>the Transport Layer delivers data to the correct application.</span>

---

# Services Provided

<span style={{ color: "#14b8a6", fontWeight: 700 }}>The Transport Layer may provide:</span>

- Reliability
- Error recovery
- Flow control
- Congestion control
- Multiplexing
- Demultiplexing

<span style={{ color: "#eab308", fontWeight: 700 }}>Different protocols provide different services.</span>

---

# Common Transport Layer Protocols

The Internet <span style={{ color: "#d946ef", fontWeight: 700 }}>mainly uses:</span>

- <span style={{ color: "#0ea5e9", fontWeight: 700 }}>TCP</span>
- <span style={{ color: "#84cc16", fontWeight: 700 }}>UDP</span>

We will study these in later chapters.

---

# TCP

TCP provides:

- Reliable communication.
- Ordered delivery.
- Error recovery.
- Flow control.
- Congestion control.

Suitable for:

- Web browsing
- Email
- Banking
- File transfer

---

# UDP

UDP provides:

- Faster communication.
- Lower overhead.
- No guaranteed delivery.

Suitable for:

- Voice calls
- Video streaming
- Online gaming
- Live broadcasting

---

# Transport Layer Data

<span style={{ color: "#fb7185", fontWeight: 700 }}>After processing,</span>

<span style={{ color: "#c084fc", fontWeight: 700 }}>the Application Layer message becomes a:</span>

```text
Segment
```

Notice the terminology.

Application Layer:

```text
Message
```

Transport Layer:

```text
Segment
```

---

# Network Layer

The Transport Layer passes the segment downward.

Now we reach:

> **Network Layer**

---

# Main Responsibility

The Network Layer is responsible for:

> <span style={{ color: "#60a5fa", fontWeight: 700 }}>Moving packets from the source host to the destination host across multiple interconnected networks.</span>

This layer answers:

> <span style={{ color: "#e879f9", fontWeight: 700 }}>Which path should the packet take?</span>

---

# Main Tasks

The Network Layer performs:

- Logical Addressing
- Routing
- Packet Forwarding

---

# IP Address

The Network Layer uses:

```text
IP Addresses
```

Examples:

```text
192.168.1.10

8.8.8.8

172.217.20.14
```

These addresses identify devices across networks.

---

# Routers Operate Here

Routers primarily work at the:

> **Network Layer**

Their main responsibility is:

```text
Receive Packet

↓

Examine Destination IP

↓

Forward Packet
```

---

# Common Network Layer Protocols

Examples:

- <span style={{ color: "#fde047", fontWeight: 700 }}>IPv4</span>
- <span style={{ color: "#fca5a5", fontWeight: 700 }}>IPv6</span>
- <span style={{ color: "#93c5fd", fontWeight: 700 }}>ICMP</span>

<span style={{ color: "#fdba74", fontWeight: 700 }}>Routing protocols such as OSPF and BGP also operate in this layer's functionality, although they serve specialized routing purposes.</span>

---

# Network Layer Data

<span style={{ color: "#86efac", fontWeight: 700 }}>After processing,</span>

<span style={{ color: "#a5b4fc", fontWeight: 700 }}>the segment becomes a:</span>

```text
Packet

or

Datagram
```

---

# Link Layer

The packet now reaches the:

> **Link Layer**

---

# Main Responsibility

The Link Layer delivers data between:

> <span style={{ color: "#38bdf8", fontWeight: 700 }}>Directly connected neighboring devices.</span>

Unlike the Network Layer,

it does **not** move data across the entire Internet.

Instead,

<span style={{ color: "#fbbf24", fontWeight: 700 }}>it moves data one hop at a time.</span>

---

# Example

```text
Laptop

↓

WiFi Router
```

<span style={{ color: "#f43f5e", fontWeight: 700 }}>This communication uses the Link Layer.</span>

---

Another example:

```text
Router A

↓

Router B
```

Again,

<span style={{ color: "#14b8a6", fontWeight: 700 }}>the Link Layer handles communication over this single physical link.</span>

---

# Main Tasks

The Link Layer performs:

- Framing
- Error Detection
- <span style={{ color: "#eab308", fontWeight: 700 }}>Medium Access Control (MAC)</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Link-Level Addressing</span>

---

# MAC Address

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Unlike the Network Layer,</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>the Link Layer uses:</span>

```text
MAC Addresses
```

Examples:

```text
00:1A:2B:3C:4D:5E
```

<span style={{ color: "#fb7185", fontWeight: 700 }}>Every network interface card has a MAC address.</span>

---

# Common Link Layer Technologies

Examples include:

- Ethernet
- WiFi (IEEE 802.11)
- PPP

<span style={{ color: "#c084fc", fontWeight: 700 }}>These technologies define how neighboring devices communicate.</span>

---

# Link Layer Data

<span style={{ color: "#60a5fa", fontWeight: 700 }}>The Network Layer packet becomes a:</span>

```text
Frame
```

Notice the terminology.

Transport:

```text
Segment
```

↓

Network:

```text
Packet
```

↓

Link:

```text
Frame
```

---

# Physical Layer

Finally,

the frame reaches the:

> **Physical Layer**

---

# Main Responsibility

The Physical Layer is responsible for:

> <span style={{ color: "#e879f9", fontWeight: 700 }}>Transmitting raw bits through the communication medium.</span>

This is the lowest layer.

---

# What Does It Actually Send?

<span style={{ color: "#fde047", fontWeight: 700 }}>The Physical Layer does not understand:</span>

- HTTP
- TCP
- IP

<span style={{ color: "#fca5a5", fontWeight: 700 }}>It simply transmits:</span>

```text
0

1

0

1

1

0

1
```

<span style={{ color: "#93c5fd", fontWeight: 700 }}>These binary digits become:</span>

- <span style={{ color: "#fdba74", fontWeight: 700 }}>Electrical signals</span>
- <span style={{ color: "#86efac", fontWeight: 700 }}>Optical signals</span>
- <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Radio waves</span>

<span style={{ color: "#38bdf8", fontWeight: 700 }}>depending on the communication medium.</span>

---

# Physical Media

Examples include:

- Copper Cable
- Fiber Optic Cable
- <span style={{ color: "#fbbf24", fontWeight: 700 }}>WiFi Radio Waves</span>
- Bluetooth
- Cellular Networks

---

# Physical Layer Data

<span style={{ color: "#f43f5e", fontWeight: 700 }}>At this layer,</span>

<span style={{ color: "#14b8a6", fontWeight: 700 }}>the information is simply:</span>

```text
Bits
```

---

its name changes.

```text

↓

Message

↓

Transport Layer

↓

Segment

↓

↓

↓

Link Layer

↓

Frame

↓

↓

Bits
```

This terminology is extremely important.

# Complete Communication Flow

Suppose you open Google's homepage.

```text
Browser

↓

Application Layer

↓

Transport Layer

↓

Network Layer

↓

Link Layer

↓

Physical Layer

↓

Internet

↓

Physical Layer

↓

Link Layer

↓

```

Each layer performs its own responsibilities.

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Layer Responsibilities Summary</span>

| Layer | Data Unit | Main Responsibility | Example Protocols |
|--------|-----------|--------------------|-------------------|
| Application | Message | Network services for applications | <span style={{ color: "#38bdf8", fontWeight: 700 }}>HTTP, HTTPS, DNS, SMTP, FTP</span> |
| Transport | Segment | End-to-end process communication | <span style={{ color: "#fbbf24", fontWeight: 700 }}>TCP, UDP</span> |
| Network | Packet (Datagram) | Routing and logical addressing | <span style={{ color: "#f43f5e", fontWeight: 700 }}>IPv4, IPv6, ICMP</span> |
| Link | Frame | Communication between neighboring devices | <span style={{ color: "#14b8a6", fontWeight: 700 }}>Ethernet, WiFi, PPP</span> |
| Physical | Bits | Transmission of signals | <span style={{ color: "#c084fc", fontWeight: 700 }}>Fiber, Copper, Radio</span> |

---

# Easy Memory Trick

Remember this order:

```text
Application

Transport

Network

Link

Physical
```

One simple mnemonic is:

> <span style={{ color: "#eab308", fontWeight: 700 }}>"All Teachers Need Loving Pupils."</span>

The first letter of each word matches the layer:

- **A** → Application
- **T** → Transport
- **N** → Network
- **L** → Link
- **P** → Physical

---

# Key Takeaways

- The Internet Protocol Stack contains five layers.
- Each layer has a specific responsibility.
- Data moves downward when sending and upward when receiving.
- The name of the data changes at each layer.
- Different protocols operate at different layers.
- Together, all five layers provide complete end-to-end communication.

---

# Technical Vocabulary

| Term | Definition |
|------|------------|
| Protocol Stack | Collection of networking layers working together to provide communication services. |
| Application Layer | Layer providing networking services to user applications. |
| Transport Layer | Layer providing logical communication between application processes. |
| Network Layer | Layer responsible for routing packets across interconnected networks. |
| Link Layer | Layer responsible for communication between directly connected devices. |
| Physical Layer | Layer responsible for transmitting raw bits over the communication medium. |
| Message | Data unit used at the Application Layer. |
| Segment | Data unit used at the Transport Layer. |
| Packet (Datagram) | Data unit used at the Network Layer. |
| Frame | Data unit used at the Link Layer. |
| Bits | Binary data transmitted by the Physical Layer. |
