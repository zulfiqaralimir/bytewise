# Section 1.4 Delay, Loss, and Throughput in Packet-Switched Networks (Part 4B)

# Processing Delay and Queueing Delay

In the previous section, we learned that every packet experiences <span style={{ color: "#38bdf8", fontWeight: 700 }}>Nodal Delay whenever it reaches a router.</span>

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Recall the equation:</span>

```text
d_nodal

=

d_proc

+

d_queue

+

d_trans

+

d_prop
```

where:

- `d_proc` = Processing Delay
- `d_queue` = Queueing Delay
- `d_trans` = Transmission Delay
- `d_prop` = Propagation Delay

In this section, <span style={{ color: "#f43f5e", fontWeight: 700 }}>we will study the first two components in detail:</span>

- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Processing Delay</span>
- <span style={{ color: "#eab308", fontWeight: 700 }}>Queueing Delay</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>These delays occur inside the router before a packet is transmitted onto the next communication link.</span>

---

# Overview of Packet Processing

Suppose a packet arrives at a router.

The router cannot immediately send it to the next destination.

Instead, the packet goes through several stages.

```text
Packet Arrives

↓

Processing

↓

Queue (if necessary)

↓

Transmission

↓

Propagation
```

The first two stages occur **inside the router**.

---

# Processing Delay (`d_proc`)

## What Is Processing Delay?

Processing Delay is:

> **The time required by a router to examine the packet, determine where it should go, and prepare it for forwarding.**

Before forwarding a packet, the router must first understand it.

---

# Real-Life Analogy

Imagine a parcel arriving at a courier office.

The employee does not immediately send it.

First, they:

- Read the destination address.
- Check whether the parcel is damaged.
- Decide which truck should carry it.
- Attach routing labels.

Only then is the parcel forwarded.

A router behaves in exactly the same way.

---

# What Happens During Processing?

When a packet reaches a router, several operations occur.

### Step 1 — Receive the Packet

The router receives the complete packet.

```text
Incoming Link

↓

Router
```

---

### Step 2 — Read the Packet Header

The router examines the **header**, not the payload.

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>The header contains important information such as:</span>

- <span style={{ color: "#84cc16", fontWeight: 700 }}>Source IP Address</span>
- <span style={{ color: "#fb7185", fontWeight: 700 }}>Destination IP Address</span>
- <span style={{ color: "#c084fc", fontWeight: 700 }}>Protocol Type</span>
- <span style={{ color: "#60a5fa", fontWeight: 700 }}>Time-to-Live (TTL)</span>
- <span style={{ color: "#e879f9", fontWeight: 700 }}>Packet Length</span>

---

### Step 3 — Error Checking

<span style={{ color: "#fde047", fontWeight: 700 }}>The router checks whether the packet has been corrupted during transmission.</span>

This is usually done using error-detection fields such as a <span style={{ color: "#fca5a5", fontWeight: 700 }}>checksum</span>.

If the packet is corrupted:

```text
Packet

↓

Discard
```

---

### Step 4 — Forwarding Table Lookup

The router searches its forwarding table.

Example:

| Destination Network | Next Hop |
|---------------------|----------|
| Network A | Link 1 |
| Network B | Link 2 |
| Network C | Link 3 |

If the destination belongs to **Network B**, the router chooses **Link 2**.

---

### Step 5 — Prepare for Transmission

After determining the outgoing link,

the router prepares the packet for transmission.

<span style={{ color: "#93c5fd", fontWeight: 700 }}>If the outgoing link is free,</span>

<span style={{ color: "#fdba74", fontWeight: 700 }}>the packet proceeds immediately.</span>

<span style={{ color: "#86efac", fontWeight: 700 }}>Otherwise,</span>

<span style={{ color: "#a5b4fc", fontWeight: 700 }}>it enters a queue.</span>

---

# Visualizing Processing Delay

```text
Packet Arrives

↓

Read Header

↓

Check Errors

↓

Forwarding Table Lookup

↓

Select Outgoing Link

↓

Ready for Transmission
```

---

# How Long Is Processing Delay?

Processing Delay is usually very small.

Modern routers are extremely powerful.

Typical values:

```text
Less than 1 millisecond
```

or even

```text
Microseconds
```

In high-performance routers,

<span style={{ color: "#38bdf8", fontWeight: 700 }}>Processing Delay is often negligible compared to other delays.</span>

---

# What Affects Processing Delay?

Several factors influence Processing Delay.

## Router Hardware

A faster processor examines packets more quickly.

---

## Router Architecture

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Specialized hardware such as ASICs (Application-Specific Integrated Circuits) can perform forwarding much faster than general-purpose CPUs.</span>

---

## Packet Complexity

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Certain packets require additional processing,</span>

<span style={{ color: "#14b8a6", fontWeight: 700 }}>such as security checks or Quality of Service (QoS) decisions.</span>

---

## Routing Table Size

Larger forwarding tables may require more <span style={{ color: "#eab308", fontWeight: 700 }}>sophisticated lookup algorithms,</span>

<span style={{ color: "#d946ef", fontWeight: 700 }}>although modern routers optimize these operations efficiently.</span>

---

# Example

Suppose a router receives:

```text
1 Million Packets per Second
```

Each packet requires:

```text
1 Microsecond
```

of processing.

The router continuously processes incoming packets before forwarding them.

---

# Important Observation

Processing Delay is generally:

- Small
- Predictable
- Nearly constant

Unlike Queueing Delay,

it does not vary dramatically under normal conditions.

---

# Queueing Delay (`d_queue`)

Now suppose the router has finished processing the packet.

Question:

> **Can the router always transmit the packet immediately?**

The answer is:

> **No.**

If another packet is already being transmitted,

the new packet must wait.

This waiting time is called:

> **Queueing Delay**

---

# What Is Queueing Delay?

Definition:

> **Queueing Delay is the time a packet spends waiting in a router's buffer before it can be transmitted.**

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Unlike Processing Delay,</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>Queueing Delay depends on network traffic.</span>

---

# Real-Life Analogy

Imagine arriving at a bank.

If no customers are waiting:

```text
You

↓

Cashier
```

You are served immediately.

Queueing Delay is:

```text
0 seconds
```

---

Now imagine:

```text
Customer 1

Customer 2

Customer 3

You
```

You must wait.

That waiting time is your Queueing Delay.

Routers behave in exactly the same way.

---

# Router Buffer

Packets waiting for transmission are stored in:

## Buffer

Definition:

> <span style={{ color: "#fb7185", fontWeight: 700 }}>A buffer is temporary memory used to store packets before transmission.</span>

Think of the buffer as the waiting room inside a router.

---

# Queue Formation

Suppose packets arrive like this:

```text
Packet A

Packet B

Packet C

Packet D
```

<span style={{ color: "#38bdf8", fontWeight: 700 }}>Only one packet can use the outgoing link.</span>

<span style={{ color: "#fbbf24", fontWeight: 700 }}>The others wait.</span>

```text
Outgoing Link

↓

Packet A (Being Sent)

Packet B (Waiting)

Packet C (Waiting)

Packet D (Waiting)
```

This waiting line is called a **queue**.

---

# Why Does Queueing Occur?

Queueing occurs whenever:

```text
Packet Arrival Rate

>

Packet Transmission Rate
```

In other words,

<span style={{ color: "#f43f5e", fontWeight: 700 }}>packets arrive faster than the router can transmit them.</span>

---

# Packet Arrival Rate

The **Packet Arrival Rate** is:

> The number of packets arriving at the router per unit time.

Example:

```text
500 packets/second
```

---

# Packet Transmission Rate

The **Packet Transmission Rate** is:

> The number of packets the router can transmit per unit time.

Example:

```text
400 packets/second
```

---

# What Happens?

If:

```text
Arrival Rate

<

Transmission Rate
```

The queue remains short.

Queueing Delay is very small.

---

If:

```text
Arrival Rate

=

Transmission Rate
```

The router operates at full capacity.

Small queues may appear.

---

If:

```text
Arrival Rate

>

Transmission Rate
```

The queue grows continuously.

Queueing Delay increases rapidly.

---

# Example

Suppose a router can transmit:

```text
100 packets/second
```

But receives:

```text
150 packets/second
```

Every second:

- 100 packets leave.
- 150 packets arrive.

Result:

```text
50 New Packets
```

remain in the queue.

Over time,

the queue becomes longer and longer.

---

# Variable Nature of Queueing Delay

Unlike Processing Delay,

Queueing Delay is unpredictable.

Sometimes:

```text
0 milliseconds
```

Sometimes:

```text
5 milliseconds
```

Sometimes:

```text
500 milliseconds
```

Sometimes:

```text
Several seconds
```

<span style={{ color: "#14b8a6", fontWeight: 700 }}>It depends entirely on network traffic.</span>

---

# Traffic Intensity

Kurose introduces an important concept called:

## Traffic Intensity

Traffic Intensity measures how heavily a communication link is being used.

It compares:

- Packet arrival rate
- Transmission capacity

A simple interpretation is:

```text
Traffic Intensity

=

Traffic Entering

/

Traffic Leaving
```

In later chapters, this is expressed mathematically using arrival rates (`a`), packet length (`L`), and transmission rate (`R`).

For now,

remember:

- <span style={{ color: "#eab308", fontWeight: 700 }}>Low Traffic Intensity → Small queues</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>High Traffic Intensity → Long queues</span>

---

# Three Traffic Conditions

## Light Traffic

```text
Packets

↓

Router

↓

Immediately Sent
```

Queue:

```text
Empty
```

Queueing Delay:

```text
Almost Zero
```

---

## Moderate Traffic

```text
Packets

↓

Router

↓

Small Queue

↓

Transmission
```

Queueing Delay:

```text
Small
```

---

## Heavy Traffic

```text
Packets

Packets

Packets

Packets

Packets
```

Router cannot keep up.

Large queue forms.

Queueing Delay becomes significant.

---

# Can Queueing Delay Become Very Large?

Yes.

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>If packets continue arriving faster than they leave,</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>the queue keeps growing.</span>

<span style={{ color: "#fb7185", fontWeight: 700 }}>Eventually:</span>

- <span style={{ color: "#c084fc", fontWeight: 700 }}>Packets wait longer.</span>
- <span style={{ color: "#60a5fa", fontWeight: 700 }}>Buffers fill.</span>
- <span style={{ color: "#e879f9", fontWeight: 700 }}>Packet Loss occurs.</span>

<span style={{ color: "#fde047", fontWeight: 700 }}>This is one reason Internet connections slow down during busy periods.</span>

---

# Processing Delay vs Queueing Delay

| Feature | Processing Delay | Queueing Delay |
|----------|------------------|----------------|
| Occurs During | Packet examination | Waiting for transmission |
| Depends On | Router hardware | Network traffic |
| Predictability | Predictable | Highly variable |
| Typical Duration | Microseconds | 0 ms to several seconds |
| Main Cause | Header processing | Busy outgoing link |

---

# Why Are These Delays Important?

Consider an online game.

If Processing Delay is:

```text
1 microsecond
```

but Queueing Delay becomes:

```text
300 milliseconds
```

The player notices lag.

Clearly,

Queueing Delay has a much greater impact.

---

# Big Picture

Every packet reaching a router experiences:

```text
Packet Arrives
        │
        ▼
Processing Delay
        │
        ▼
Queueing Delay
        │
        ▼
Transmission Delay
        │
        ▼
Propagation Delay
        │
        ▼
Next Router
```

<span style={{ color: "#fca5a5", fontWeight: 700 }}>The first two stages occur entirely inside the router.</span>

---

# Key Takeaways

- Processing Delay is the time required to examine and prepare a packet for forwarding.
- Processing Delay is usually very small and nearly constant.
- Queueing Delay is the time a packet waits inside the router's buffer.
- Queueing Delay depends on network traffic.
- Buffers temporarily store packets waiting for transmission.
- Heavy traffic increases Queueing Delay.
- If traffic continues increasing, Packet Loss may occur.

---

# Technical Vocabulary

| Term | Definition |
|------|------------|
| Processing Delay (`d_proc`) | Time required for a router to examine, verify, and prepare a packet for forwarding. |
| Queueing Delay (`d_queue`) | Time a packet spends waiting in a router's buffer before transmission. |
| Buffer | Temporary memory inside a router used to store packets awaiting transmission. |
| Queue | Ordered list of packets waiting to use an outgoing communication link. |
| Packet Arrival Rate | Number of packets arriving at a router per unit time. |
| Packet Transmission Rate | Number of packets a router can transmit per unit time. |
| Traffic Intensity | Measure of how heavily a communication link is utilized relative to its capacity. |
| Forwarding Table | Data structure used by routers to determine the next hop for packets. |
| Checksum | Error-detection value used to identify packet corruption during transmission. |
| ASIC (Application-Specific Integrated Circuit) | Specialized hardware used in routers to accelerate packet forwarding. |
