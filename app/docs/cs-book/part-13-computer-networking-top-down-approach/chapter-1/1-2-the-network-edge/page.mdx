# 1.2 The Network Edge

After understanding what the Internet is, we now move to an important part of the Internet architecture:

> The Network Edge

<span className="rev-concept">The Internet can be divided into two major parts:</span>

```text
Internet
│
├── Network Edge
│
└── Network Core
```

<span className="rev-key">The Network Edge consists of:</span>

- End Systems (Hosts)
- Access Networks
- Physical Media

This is where users and applications interact with the Internet.

---

# What Is the Network Edge?

The Network Edge is the collection of devices located at the boundary of the Internet.

Examples:

- Desktop Computers
- Laptops
- Smartphones
- Web Servers
- Cloud Servers
- Smart TVs
- IoT Devices

<span className="rev-key">These devices are called:</span>

## <span className="rev-key">Hosts</span>

or

## <span className="rev-key">End Systems</span>

because communication starts and ends at these devices.

Example:

```text
Alice's Laptop
      ↓
Internet
      ↓
Google Server
```

<span className="rev-key">Both devices are located at the Network Edge.</span>

---

# Role of the Network Edge

The Network Edge is responsible for:

- Running applications
- Generating data
- Receiving data
- Communicating with other hosts

Examples:

| Application | Communication |
|------------|--------------|
| Web Browser | Browser ↔ Web Server |
| WhatsApp | User ↔ WhatsApp Server |
| Zoom | User ↔ Zoom Servers |
| Netflix | User ↔ Content Servers |

The Network Edge is where users experience Internet services.

---

# <span className="rev-key">Client-Server Architecture</span>

Many Internet applications use:

## Client-Server Architecture

In this model:

### Client

Requests services.

Examples:

- Browser
- Mobile App
- Desktop Application

### Server

Provides services.

Examples:

- Google Server
- Netflix Server
- Facebook Server

Example:

```text
Client
   ↓ Request
Server
   ↑ Response
```

---

## Characteristics of Servers

Servers usually have:

- High-performance hardware
- Large storage capacity
- Powerful processors
- <span className="rev-key">High-speed network connections</span>

<span className="rev-key">Servers are typically located in:</span>

### <span className="rev-key">Data Centers</span>

Examples:

- Google Data Centers
- Amazon AWS
- Microsoft Azure

---

# Peer-to-Peer (P2P) Architecture

<span className="rev-concept">Not all applications use client-server architecture.</span>

Another model is:

## Peer-to-Peer (P2P)

In P2P systems:

- <span className="rev-key">No permanently active server exists.</span>
- <span className="rev-key">Peers communicate directly.</span>
- <span className="rev-key">Every device can act as both client and server.</span>

Example:

```text
Peer A ↔ Peer B
    ↕
Peer C ↔ Peer D
```

---

## Advantages of P2P

### Self-Scalability

<span className="rev-key">As more users join:</span>

- <span className="rev-key">More resources become available.</span>

### Cost Efficient

<span className="rev-key">Less dependence on expensive centralized servers.</span>

---

## <span className="rev-key">Examples of P2P Applications</span>

- BitTorrent
- Early Skype
- <span className="rev-key">Blockchain Networks</span>

---

# Processes Communicating

Applications running on hosts are called:

## Processes

Examples:

- Browser Process
- WhatsApp Process
- Zoom Process

<span className="rev-key">Communication actually occurs between processes.</span>

Example:

```text
Browser Process
        ↓
Internet
        ↓
Web Server Process
```

---

# Client Process and Server Process

A process that initiates communication is called:

## Client Process

A process waiting for communication is called:

## Server Process

Example:

```text
Browser Process
      ↓
Web Server Process
```

Browser = Client Process

Web Server = Server Process

---

# Access Networks

To participate in the Internet, hosts require connectivity.

<span className="rev-key">The network that connects end systems to the first router is called:</span>

## Access Network

Think of an Access Network as:

> <span className="rev-concept">The bridge between a user and the Internet.</span>

---

# Types of Access Networks

The book discusses <span className="rev-key">several major access technologies</span>.

---

# 1. Residential Access Networks

<span className="rev-key">Used in homes.</span>

Examples:

- Fiber-to-the-Home (FTTH)
- Cable Internet
- DSL

---

## DSL (Digital Subscriber Line)

<span className="rev-key">DSL uses existing telephone lines.</span>

Example:

```text
Home
  ↓
Telephone Line
  ↓
Telephone Company
  ↓
Internet
```

---

### Characteristics of DSL

- Uses copper telephone wires.
- <span className="rev-key">Internet and telephone can operate simultaneously.</span>
- Speed depends on distance from telephone company.

---

### Advantages

- Existing infrastructure.
- Widely available.

### Disadvantages

- Lower speeds compared to fiber.
- Performance decreases with distance.

---

# Cable Internet Access

<span className="rev-key">Cable Internet uses television cable infrastructure.</span>

Example:

```text
Home
  ↓
Cable Network
  ↓
ISP
  ↓
Internet
```

---

## Characteristics

Uses:

### Hybrid Fiber Coaxial (HFC)

Combination of:

- Fiber Optic Cable
- Coaxial Cable

---

### Advantages

- Higher speed than DSL.
- Widely deployed.

### Disadvantages

- Shared bandwidth.
- Performance may decrease during peak hours.

---

# Fiber-to-the-Home (FTTH)

FTTH provides Internet directly through fiber-optic cables.

Example:

```text
Home
   ↓
Fiber Cable
   ↓
ISP
```

---

## Advantages

- Extremely high speed.
- Very low latency.
- High reliability.

---

## Typical Speeds

- 100 Mbps
- 1 Gbps
- 10 Gbps

---

## Why Fiber Is Important

Fiber is becoming the preferred access technology because it provides:

- High bandwidth
- Future scalability
- Better reliability

---

# 2. Enterprise Access Networks

Used in:

- Universities
- Offices
- Corporations

Examples:

### <span className="rev-key">Ethernet</span>

### <span className="rev-key">WiFi</span>

---

# Ethernet

Ethernet is the most widely used <span className="rev-key">wired LAN technology</span>.

Example:

```text
Computer
    ↓
Ethernet Switch
    ↓
Router
```

---

## Ethernet Characteristics

- High speed
- Reliable
- Low latency
- Cost effective

Common speeds:

- 100 Mbps
- 1 Gbps
- 10 Gbps

---

# WiFi Access Networks

WiFi provides wireless connectivity.

Based on:

### IEEE 802.11 Standards

Example:

```text
Laptop
   ))))
WiFi Access Point
      ↓
Router
      ↓
Internet
```

---

## Advantages

- Mobility
- Easy installation
- No cables required

---

## Disadvantages

- Interference
- Limited range
- Lower reliability than wired networks

---

# 3. Mobile Access Networks

Used by smartphones and tablets.

Examples:

- 3G
- 4G LTE
- 5G

Example:

```text
Phone
   ↓
Cell Tower
   ↓
Mobile Network
   ↓
Internet
```

---

## Benefits

- Wide coverage
- Mobility
- Internet access anywhere

---

## Challenges

- Signal quality variations
- Congestion
- Coverage limitations

---

# Network Performance Metrics

<span className="rev-key">Access networks are often evaluated using:</span>

---

## Bandwidth

Bandwidth refers to:

> The maximum amount of data that can be transmitted per unit time.

Example:

```text
100 Mbps Link
```

Can theoretically transfer:

100 million bits per second.

---

## Throughput

<span className="rev-key">Throughput refers to:</span>

> <span className="rev-key">The actual data transfer rate achieved.</span>

Example:

Bandwidth:

```text
100 Mbps
```

Actual Transfer:

```text
80 Mbps
```

Throughput = 80 Mbps

---

## Latency (Delay)

Latency is:

> The time required for data to travel from source to destination.

Lower latency means:

- Faster response
- Better gaming
- Better video calls

---

# 1.2.2 Physical Media

The next question is:

> How does data physically travel?

The answer is:

Through Physical Media.

---

# What Is Physical Media?

Physical Media refers to:

> <span className="rev-key">The actual medium through which electromagnetic signals travel.</span>

Examples:

- Copper Wire
- Fiber Optic Cable
- Wireless Radio Waves

---

# Guided Media

<span className="rev-key">Signals follow a physical path.</span>

Examples:

### Twisted Pair Copper Cable

### Coaxial Cable

### Fiber Optic Cable

---

# Twisted Pair Copper Cable

Most common wired medium.

Example:

```text
Computer
   ↓
Ethernet Cable
   ↓
Switch
```

---

## Characteristics

- Inexpensive
- Easy installation
- Widely available

Used in:

- Ethernet LANs
- Telephone Systems

---

# Coaxial Cable

Used in:

- Cable Television
- Cable Internet

Structure:

```text
Central Copper Conductor
      ↓
Insulation
      ↓
Shielding
```

---

## Advantages

- Better shielding
- Higher bandwidth than twisted pair

---

# Fiber Optic Cable

<span className="rev-key">Fiber carries data using:</span>

### <span className="rev-key">Light Pulses</span>

<span className="rev-key">instead of electrical signals.</span>

---

## How Fiber Works

```text
Laser
   ↓
Light Pulse
   ↓
Fiber Cable
   ↓
Receiver
```

---

## Advantages

### Extremely High Bandwidth

Can support:

- Gigabits per second
- Terabits per second

### Low Signal Loss

### <span className="rev-key">Immune to Electromagnetic Interference</span>

### Long Distance Transmission

---

## Disadvantages

- Higher installation cost
- More complex deployment

---

# Unguided Media

<span className="rev-key">Signals travel freely through space.</span>

Examples:

### Radio Waves

### Microwave Signals

### Satellite Communication

---

# Radio Channels

Used by:

- <span className="rev-key">WiFi</span>
- <span className="rev-key">Bluetooth</span>
- <span className="rev-key">Cellular Networks</span>

Characteristics:

- Easy deployment
- Supports mobility

Challenges:

- Interference
- Signal attenuation
- <span className="rev-key">Security concerns</span>

---

# Satellite Communication

Used when terrestrial infrastructure is unavailable.

Example:

```text
User
   ↓
Satellite
   ↓
Ground Station
   ↓
Internet
```

---

## Advantages

- Global coverage
- Remote-area connectivity

---

## Disadvantages

- <span className="rev-key">High latency</span>
- <span className="rev-key">Higher cost</span>

---
