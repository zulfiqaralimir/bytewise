# Section 1.6 Networks Under Attack (Part 6A)

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Introduction to Network Security</span>

In the previous sections, we learned:

- How the Internet works.
- How packets travel through networks.
- The Internet Protocol Stack.
- Delay, Packet Loss, and Throughput.
- Protocol Layers.

Until now, we have assumed something very important:

> <span style={{ color: "#38bdf8", fontWeight: 700 }}>Every device in the network behaves honestly.</span>

But in the real world,

this assumption is not true.

The Internet contains:

- Hackers
- Cybercriminals
- Malicious Software
- Nation-State Attackers
- Organized Crime Groups
- Insider Threats

These attackers constantly attempt to:

- Steal information
- Modify data
- Interrupt communication
- Gain unauthorized access
- Damage systems

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Therefore,</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>modern networking is not just about communication.</span>

<span style={{ color: "#14b8a6", fontWeight: 700 }}>It is also about:</span>

> <span style={{ color: "#eab308", fontWeight: 700 }}>Security</span>

---

# Why Is Network Security Important?

Imagine using online banking.

When you transfer money,

you expect:

- Only your bank to receive the information.
- Nobody to change the transaction.
- The banking service to remain available.

If any of these fail,

serious problems occur.

The same applies to:

- Email
- Social Media
- Cloud Storage
- Government Systems
- Hospitals
- Power Grids
- Military Networks

Without security,

the Internet would not be trustworthy.

---

# Real-Life Analogy

Imagine sending a confidential letter through postal mail.

You expect:

- Nobody opens the envelope.
- Nobody changes the letter.
- The letter reaches the recipient.

Networking security has exactly the same goals.

---

# What Is Network Security?

Definition:

> **Network Security is the practice of protecting computer networks, devices, applications, and data from unauthorized access, attacks, misuse, modification, or destruction.**

Notice that network security protects:

- Data
- Devices
- Communication
- Services

---

# The Three Fundamental Goals of Security

<span style={{ color: "#d946ef", fontWeight: 700 }}>Almost every security mechanism is designed to protect three fundamental properties.</span>

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>These are known as the:</span>

> <span style={{ color: "#84cc16", fontWeight: 700 }}>CIA Triad</span>

The CIA Triad stands for:

- Confidentiality
- Integrity
- Availability

<span style={{ color: "#fb7185", fontWeight: 700 }}>This is one of the most important concepts in cybersecurity.</span>

---

# Visualizing the CIA Triad

```text
            Network Security

                   │

      ┌────────────┼────────────┐

      ▼            ▼            ▼

Confidentiality Integrity Availability
```

<span style={{ color: "#c084fc", fontWeight: 700 }}>Almost every attack attempts to violate one or more of these principles.</span>

---

# Confidentiality

## What Is Confidentiality?

Definition:

> **Confidentiality ensures that information is accessible only to authorized users.**

Simply put,

<span style={{ color: "#60a5fa", fontWeight: 700 }}>only the intended recipient should be able to read the data.</span>

---

# Real-Life Analogy

Suppose you send a sealed envelope.

```text
You

↓

Sealed Envelope

↓

Friend
```

Only your friend should read the contents.

If someone secretly opens the envelope,

confidentiality has been violated.

---

# Networking Example

Suppose you log into:

```text
https://www.bank.com
```

Your browser sends:

- Username
- Password

If a hacker captures your password,

confidentiality has been compromised.

---

# How Is Confidentiality Achieved?

<span style={{ color: "#e879f9", fontWeight: 700 }}>Several technologies help maintain confidentiality.</span>

Examples include:

- <span style={{ color: "#fde047", fontWeight: 700 }}>Encryption</span>
- <span style={{ color: "#93c5fd", fontWeight: 700 }}>Authentication</span>
- <span style={{ color: "#86efac", fontWeight: 700 }}>Secure Communication Protocols</span>
- <span style={{ color: "#fca5a5", fontWeight: 700 }}>Access Control</span>

We will study these in later chapters.

---

# Integrity

## What Is Integrity?

Definition:

> **Integrity ensures that data is not altered during storage or transmission without authorization.**

<span style={{ color: "#fdba74", fontWeight: 700 }}>The receiver should obtain:</span>

<span style={{ color: "#38bdf8", fontWeight: 700 }}>Exactly the same data</span>

<span style={{ color: "#fbbf24", fontWeight: 700 }}>that the sender transmitted.</span>

---

# Real-Life Analogy

Suppose you write:

```text
Transfer

$100
```

Someone secretly changes it to:

```text
Transfer

$10,000
```

The message has been modified.

Integrity has been violated.

---

# Networking Example

Suppose a hospital sends a patient's medical record.

Original:

```text
Blood Group

A+
```

An attacker changes it to:

```text
Blood Group

O-
```

This modification could be life-threatening.

Integrity protects against such unauthorized changes.

---

# How Is Integrity Achieved?

Common techniques include:

- <span style={{ color: "#f43f5e", fontWeight: 700 }}>Cryptographic Hash Functions</span>
- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Checksums</span>
- <span style={{ color: "#eab308", fontWeight: 700 }}>Digital Signatures</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Message Authentication Codes (MACs)</span>

---

# Availability

## What Is Availability?

Definition:

> **Availability ensures that authorized users can access systems and services whenever they are needed.**

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Even perfectly confidential data is useless if users cannot access it.</span>

---

# Real-Life Analogy

Suppose your bank is perfectly secure.

But every ATM is offline.

Customers cannot withdraw money.

The service is unavailable.

Availability has been lost.

---

# Networking Example

Suppose a company's website crashes because millions of fake requests overload the server.

Customers cannot access the website.

Availability has been compromised.

---

# How Is Availability Achieved?

Examples include:

- <span style={{ color: "#84cc16", fontWeight: 700 }}>Redundant Servers</span>
- <span style={{ color: "#fb7185", fontWeight: 700 }}>Backup Systems</span>
- <span style={{ color: "#c084fc", fontWeight: 700 }}>Load Balancers</span>
- <span style={{ color: "#60a5fa", fontWeight: 700 }}>Fault Tolerance</span>
- <span style={{ color: "#e879f9", fontWeight: 700 }}>DDoS Protection</span>

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Understanding the CIA Triad Together</span>

Suppose you send an email.

```text
Sender

↓

Internet

↓

Receiver
```

Security requires:

### Confidentiality

Only the receiver can read it.

---

### Integrity

Nobody changes the email.

---

### Availability

The email service remains operational.

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Visual Example</span>

```text
Message

↓

Encryption

↓

Secure Network

↓

Integrity Verification

↓

Receiver

↓

Available Service
```

All three principles work together.

---

# Who Attacks Networks?

Another important question arises.

> **Who performs cyber attacks?**

Not all attackers are the same.

Different attackers have different goals.

---

# Types of Attackers

Common categories include:

- Individual Hackers
- Cybercriminals
- Organized Crime Groups
- Nation-State Attackers
- Terrorist Organizations
- Insider Threats
- Script Kiddies

Let's study each one.

---

# Individual Hackers

Some individuals attack systems:

- For curiosity.
- To learn.
- For recognition.
- To demonstrate skills.

Not every hacker has malicious intentions.

Some become ethical hackers or security researchers.

---

# Cybercriminals

Cybercriminals attack for:

- Financial gain.
- Identity theft.
- Credit card fraud.
- Ransom payments.

<span style={{ color: "#fde047", fontWeight: 700 }}>Their primary motivation is profit.</span>

---

# Organized Crime Groups

These are professional criminal organizations.

They often operate internationally.

Activities include:

- Ransomware campaigns.
- Banking fraud.
- Cryptocurrency theft.
- Data theft.

---

# Nation-State Attackers

Some governments conduct cyber operations against other countries.

Objectives may include:

- Espionage.
- Intelligence gathering.
- Military operations.
- Infrastructure disruption.

These attackers usually possess significant technical resources.

---

# Insider Threats

Not every attacker comes from outside the organization.

Sometimes,

employees themselves misuse their access.

Examples include:

- Stealing confidential data.
- Deleting files.
- Selling customer information.
- Sabotaging systems.

These are called:

> **Insider Threats**

---

# Script Kiddies

A Script Kiddie is:

> **A person with limited technical knowledge who uses tools created by others to perform attacks.**

<span style={{ color: "#93c5fd", fontWeight: 700 }}>They often download ready-made attack software without understanding how it works.</span>

---

# Motivations of Attackers

Attackers may be motivated by:

- Money
- Politics
- Revenge
- Curiosity
- Competition
- Fame
- Ideology

Understanding motivation helps security professionals design appropriate defenses.

---

# Passive and Active Attacks

Attacks are commonly divided into two major categories.

```text
Cyber Attacks

│

├── Passive Attacks

└── Active Attacks
```

---

# Passive Attack

Definition:

> **A Passive Attack attempts to** <span style={{ color: "#86efac", fontWeight: 700 }}>observe or collect information without modifying the communication.</span>

The attacker remains hidden.

<span style={{ color: "#fca5a5", fontWeight: 700 }}>They simply listen.</span>

---

# Real-Life Analogy

Imagine someone secretly listening to your conversation.

They do not interrupt.

They simply collect information.

This is a passive attack.

---

# Networking Example

Suppose an attacker captures network packets.

They read:

- Usernames
- Passwords
- Emails

But they do not modify anything.

This is a passive attack.

---

# Characteristics of Passive Attacks

Passive attacks:

- <span style={{ color: "#fdba74", fontWeight: 700 }}>Observe communication.</span>
- Collect information.
- Do not alter data.
- <span style={{ color: "#38bdf8", fontWeight: 700 }}>Are difficult to detect.</span>

Victims often never realize they were monitored.

---

# Active Attack

Definition:

> **An Active Attack attempts to modify, disrupt, inject, or destroy communication or services.**

Unlike passive attacks,

active attacks interfere with normal network operation.

---

# Real-Life Analogy

Imagine someone intercepts your letter,

changes its contents,

and sends it onward.

This is an active attack.

---

# Networking Example

Suppose an attacker:

- Modifies packets.
- Deletes packets.
- Injects fake packets.
- Disrupts communication.

These are active attacks.

---

# Characteristics of Active Attacks

Active attacks:

- Modify communication.
- Interrupt services.
- Corrupt data.
- <span style={{ color: "#fbbf24", fontWeight: 700 }}>Are usually easier to detect.</span>

---

# Passive vs Active Attacks

| Passive Attack | Active Attack |
|----------------|---------------|
| Observes communication | Modifies communication |
| Collects information | Changes or destroys information |
| Difficult to detect | Often easier to detect |
| Does not affect system operation | Interrupts normal operation |
| <span style={{ color: "#f43f5e", fontWeight: 700 }}>Example: Packet Sniffing</span> | <span style={{ color: "#14b8a6", fontWeight: 700 }}>Example: Denial-of-Service</span> |

---


---

# Why Study Network Attacks?

Understanding attacks helps us:

- <span style={{ color: "#eab308", fontWeight: 700 }}>Build secure systems.</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Detect suspicious behavior.</span>
- <span style={{ color: "#0ea5e9", fontWeight: 700 }}>Design better protocols.</span>
- <span style={{ color: "#84cc16", fontWeight: 700 }}>Protect sensitive information.</span>

<span style={{ color: "#fb7185", fontWeight: 700 }}>Security professionals must understand:</span>

> <span style={{ color: "#c084fc", fontWeight: 700 }}>How attackers think</span>

before they can defend against them.

---

# Key Takeaways

- Modern networks face many different cyber threats.
- The primary goals of network security are represented by the CIA Triad:
  - Confidentiality
  - Integrity
  - Availability
- Attackers include individuals, cybercriminals, nation-states, insiders, and organized groups.
- Passive attacks observe communication without changing it.
- Active attacks modify, interrupt, or destroy communication.
- Understanding attacker behavior is essential for designing secure networks.

---

# Technical Vocabulary

| Term | Definition |
|------|------------|
| Network Security | Protection of networks, systems, and data from unauthorized access and attacks. |
| CIA Triad | Security model consisting of Confidentiality, Integrity, and Availability. |
| Confidentiality | Ensuring that information is accessible only to authorized users. |
| Integrity | Ensuring that information is not modified without authorization. |
| Availability | Ensuring that systems and services remain accessible to authorized users. |
| Cybercriminal | Individual or group that conducts cyber attacks for financial or criminal gain. |
| Nation-State Attacker | Government-sponsored attacker conducting cyber operations for political or military objectives. |
| Insider Threat | Authorized user who intentionally or unintentionally compromises security. |
| Passive Attack | Attack that observes or captures information without modifying it. |
| Active Attack | Attack that modifies, disrupts, injects, or destroys communication. |
| Script Kiddie | Inexperienced attacker who uses pre-built hacking tools created by others. |
