# Section 1.5 Protocol Layers and Their Service Models (Part 5D)

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>The OSI Reference Model and Service Models</span>

In the previous parts, we studied:

- Why networking is divided into layers.
- The Internet Protocol Stack (5-layer model).
- Encapsulation and Decapsulation.

Now another important question arises.

> **If the Internet uses the Five-Layer Protocol Stack, why do networking books also teach the OSI Model?**

This is one of the most common questions asked by beginners.

To answer it, we must understand the **OSI Reference Model**.

---

# What Is the OSI Model?

OSI stands for:

> <span style={{ color: "#38bdf8", fontWeight: 700 }}>Open Systems Interconnection</span>

It is a **reference model** developed by the <span style={{ color: "#fbbf24", fontWeight: 700 }}>International Organization for Standardization (ISO)</span> to <span style={{ color: "#f43f5e", fontWeight: 700 }}>standardize how different computer systems communicate.</span>

Notice the phrase:

> **Reference Model**

The OSI model is primarily used to:

- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Explain networking concepts.</span>
- <span style={{ color: "#eab308", fontWeight: 700 }}>Provide a common vocabulary.</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Help engineers design protocols.</span>

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>It is not the protocol stack actually used by the Internet.</span>

---

# Why Was the OSI Model Created?

Before standardization,

different manufacturers built networking equipment that often could not communicate with one another.

For example:

- IBM created its own networking protocols.
- DEC created another.
- Xerox created another.

This caused compatibility problems.

The OSI model provided a common framework so that networking technologies could be designed consistently.

---

# The Seven Layers of the OSI Model

Unlike the Internet Protocol Stack,

the OSI model contains **seven layers**.

```text
+---------------------------+
|     Application Layer     |
+---------------------------+
|    Presentation Layer     |
+---------------------------+
|      Session Layer        |
+---------------------------+
|     Transport Layer       |
+---------------------------+
|      Network Layer        |
+---------------------------+
|        Data Link Layer    |
+---------------------------+
|      Physical Layer       |
+---------------------------+
```

This diagram is one of the most famous in networking.

---

# Layer Order

From top to bottom:

```text
Application

↓

Presentation

↓

Session

↓

Transport

↓

Network

↓

Data Link

↓

Physical
```

Data travels downward when sending,

and upward when receiving.

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Overview of Each Layer</span>

| Layer | Main Responsibility |
|--------|---------------------|
| Application | Provides services to user applications. |
| <span style={{ color: "#84cc16", fontWeight: 700 }}>Presentation</span> | <span style={{ color: "#84cc16", fontWeight: 700 }}>Data representation, encryption, compression.</span> |
| <span style={{ color: "#fb7185", fontWeight: 700 }}>Session</span> | <span style={{ color: "#fb7185", fontWeight: 700 }}>Establishes, manages, and terminates communication sessions.</span> |
| Transport | End-to-end communication between processes. |
| Network | Routing and logical addressing. |
| Data Link | Communication between neighboring devices. |
| Physical | Transmission of bits. |

Notice that the Internet Protocol Stack combines some of these responsibilities.

---

# Physical Layer

The lowest layer.

Main responsibility:

> Transmit raw bits through the communication medium.

Examples:

- Copper Cable
- Fiber Optic Cable
- Radio Waves

Data Unit:

```text
Bits
```

---

# Data Link Layer

Main responsibility:

Reliable communication between directly connected devices.

Functions include:

- Framing
- Error Detection
- MAC Addressing
- Medium Access Control

Examples:

- Ethernet
- WiFi
- PPP

Data Unit:

```text
Frame
```

---

# Network Layer

Main responsibility:

Move packets between different networks.

Functions:

- Routing
- IP Addressing
- Packet Forwarding

Examples:

- IPv4
- IPv6
- ICMP

Data Unit:

```text
Packet
```

---

# Transport Layer

Main responsibility:

Provide end-to-end communication between application processes.

<span style={{ color: "#c084fc", fontWeight: 700 }}>Functions:</span>

- <span style={{ color: "#60a5fa", fontWeight: 700 }}>Reliability</span>
- <span style={{ color: "#e879f9", fontWeight: 700 }}>Flow Control</span>
- <span style={{ color: "#fde047", fontWeight: 700 }}>Congestion Control</span>
- <span style={{ color: "#93c5fd", fontWeight: 700 }}>Error Recovery</span>

<span style={{ color: "#86efac", fontWeight: 700 }}>Examples:</span>

- <span style={{ color: "#fca5a5", fontWeight: 700 }}>TCP</span>
- <span style={{ color: "#fdba74", fontWeight: 700 }}>UDP</span>

Data Unit:

```text
Segment
```

---

# Session Layer

The Session Layer is one of the major differences between the OSI model and the Internet Protocol Stack.

Its responsibility is:

> **Managing communication sessions between applications.**

---

# What Is a Session?

A session is:

> <span style={{ color: "#38bdf8", fontWeight: 700 }}>An ongoing exchange of information</span> between two communicating applications.

Examples:

- Video Conference
- Remote Login
- Database Connection

The Session Layer helps:

- Start sessions.
- Maintain sessions.
- End sessions.

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Real-Life Analogy</span>

Imagine a Zoom meeting.

Someone must:

- Create the meeting.
- Allow participants to join.
- End the meeting.

These responsibilities resemble the Session Layer.

---

# Presentation Layer

The Presentation Layer prepares data so that different systems can understand it.

Its responsibilities include:

- <span style={{ color: "#fbbf24", fontWeight: 700 }}>Data Translation</span>
- <span style={{ color: "#f43f5e", fontWeight: 700 }}>Encryption</span>
- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Decryption</span>
- <span style={{ color: "#eab308", fontWeight: 700 }}>Compression</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Decompression</span>

---

# Why Is It Needed?

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Suppose one computer stores text using one character encoding,</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>while another uses a different encoding.</span>

The Presentation Layer ensures both computers interpret the data correctly.

---

# Examples

Encryption:

```text
Plain Text

↓

Encrypted Text
```

Compression:

```text
Large File

↓

Smaller File
```

<span style={{ color: "#fb7185", fontWeight: 700 }}>Translation:</span>

```text
ASCII

↓

Unicode
```

---

# Application Layer

The highest layer.

Provides network services directly to user applications.

Examples:

- HTTP
- HTTPS
- FTP
- SMTP
- DNS

This layer is similar to the Application Layer in the Internet Protocol Stack.

---

Now let's compare both architectures.

---


---

```text
Application

↓

```

Notice:

The OSI model has **two additional layers**:

- Presentation

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Mapping Between the Models</span>

| Internet Protocol Stack | OSI Model |
|--------------------------|-----------|
| Application | Application + Presentation + Session |
| Transport | Transport |
| Network | Network |
| Link | Data Link |
| Physical | Physical |

The Internet combines the responsibilities of the Application, Presentation, and Session Layers into a single Application Layer.

---

# Why Doesn't the Internet Use the Full OSI Model?

The OSI model is <span style={{ color: "#38bdf8", fontWeight: 700 }}>excellent for teaching networking concepts.</span>

However,

<span style={{ color: "#fbbf24", fontWeight: 700 }}>the Internet evolved differently.</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Many Presentation and Session Layer functions are handled by:</span>

- <span style={{ color: "#14b8a6", fontWeight: 700 }}>Applications themselves.</span>
- <span style={{ color: "#eab308", fontWeight: 700 }}>Application Layer protocols.</span>
- <span style={{ color: "#d946ef", fontWeight: 700 }}>Libraries within operating systems.</span>

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Examples:</span>

<span style={{ color: "#84cc16", fontWeight: 700 }}>Encryption is often handled by:</span>

```text
TLS

SSL
```

<span style={{ color: "#fb7185", fontWeight: 700 }}>Compression is often handled by:</span>

- <span style={{ color: "#c084fc", fontWeight: 700 }}>Web browsers.</span>
- <span style={{ color: "#60a5fa", fontWeight: 700 }}>File transfer applications.</span>
- <span style={{ color: "#e879f9", fontWeight: 700 }}>Multimedia software.</span>

Therefore,

<span style={{ color: "#fde047", fontWeight: 700 }}>dedicated Presentation and Session Layers are usually unnecessary in modern Internet implementations.</span>

---

# <span style={{ color: "#a5b4fc", fontWeight: 700 }}>Why Is the OSI Model Still Important?</span>

Even though the Internet does not fully implement the OSI model,

it remains extremely valuable.

Reasons include:

- Standard networking terminology.
- Easier learning.
- Vendor documentation.
- Certification exams.
- Network troubleshooting.

Most networking professionals use OSI terminology daily.

---

# Internet Protocol Stack vs OSI Model

| Feature | Internet Protocol Stack | OSI Model |
|---------|-------------------------|-----------|
| Number of Layers | 5 | 7 |
| Used by the Internet | Yes | No (primarily conceptual) |
| Practical Implementation | Yes | Mostly educational/reference |
| Developed By | DARPA / Internet Community | ISO |
| Simplicity | Simpler | More detailed |
| Flexibility | High | High |

---

# Service Models

Now let's study another important concept introduced in this section.

> **Service Model**

---

# What Is a Service?

<span style={{ color: "#86efac", fontWeight: 700 }}>Recall what we learned earlier.</span>

<span style={{ color: "#fca5a5", fontWeight: 700 }}>Every layer provides services to the layer above it.</span>

Example:

```text
Application Layer

↓

Requests Communication

↓

Transport Layer
```

The Transport Layer provides communication services.

---

# Definition

A **Service** is:

> **The functionality that one protocol layer offers to the layer directly above it.**

<span style={{ color: "#fdba74", fontWeight: 700 }}>Notice:</span>

<span style={{ color: "#38bdf8", fontWeight: 700 }}>A service describes:</span>

> <span style={{ color: "#fbbf24", fontWeight: 700 }}>What a layer provides</span>

<span style={{ color: "#f43f5e", fontWeight: 700 }}>It does not necessarily describe how the layer implements it.</span>

---

# Real-Life Analogy

Imagine electricity.

You plug your laptop into a wall socket.

You expect:

```text
Electric Power
```

You do not need to understand:

- Power plants.
- Transformers.
- Transmission lines.

The wall socket provides a service.

Similarly,

network layers provide services while hiding implementation details.

---

# Example

The Transport Layer may provide:

```text
Reliable Delivery
```

The Application Layer simply uses that service.

It does not need to understand:

- Retransmissions.
- Sequence Numbers.
- Acknowledgments.

These implementation details remain hidden.

---

# Advantages of Service Models

Service models provide several benefits.

---

## Abstraction

Higher layers ignore lower-level implementation details.

---

## Modularity

Each layer can evolve independently.

---

## Easier Development

<span style={{ color: "#14b8a6", fontWeight: 700 }}>Application developers need not understand routing or physical transmission.</span>

---

## Standardization

<span style={{ color: "#eab308", fontWeight: 700 }}>Common service definitions improve interoperability.</span>

---

# Layer Responsibilities Summary

| Layer | Primary Service |
|--------|-----------------|
| Application | Network services for applications |
| Transport | End-to-end process communication |
| Network | Host-to-host packet delivery across networks |
| Link | Communication between neighboring devices |
| Physical | Transmission of raw bits |

---

<span style={{ color: "#d946ef", fontWeight: 700 }}>Every layer depends upon the services of the layer below it.</span>

# Common Misconceptions

## Misconception 1

> The OSI Model is the protocol stack used by the Internet.

Incorrect.

The Internet primarily uses the **Internet Protocol Stack (TCP/IP model).**

---

## Misconception 2

> The Presentation Layer always performs encryption.

Incorrect.

Many modern applications perform encryption themselves using protocols such as TLS.

---

## Misconception 3

> Every layer communicates directly with every other layer.

Incorrect.

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>Each layer communicates only with:</span>

- <span style={{ color: "#84cc16", fontWeight: 700 }}>The layer immediately above.</span>
- <span style={{ color: "#fb7185", fontWeight: 700 }}>The layer immediately below.</span>
- <span style={{ color: "#c084fc", fontWeight: 700 }}>Its logical peer layer on the remote host.</span>

---

## Misconception 4

> Service and Protocol mean the same thing.

Incorrect.

<span style={{ color: "#60a5fa", fontWeight: 700 }}>A service describes what a layer offers.</span>

<span style={{ color: "#e879f9", fontWeight: 700 }}>A protocol defines how two peer entities implement that service.</span>

---

# Key Takeaways

- The OSI Reference Model contains seven layers.
- The Internet Protocol Stack contains five layers.
- The Internet combines the OSI Presentation and Session responsibilities into the Application Layer.
- The OSI model is mainly used for education, standardization, and troubleshooting.
- Every layer provides services to the layer above it.
- Service models hide implementation details and simplify network design.

---

# Technical Vocabulary

| Term | Definition |
|------|------------|
| OSI (Open Systems Interconnection) Model | A seven-layer reference model developed by ISO for standardizing network communication concepts. |
| Presentation Layer | OSI layer responsible for data representation, encryption, compression, and translation. |
| Session Layer | OSI layer responsible for establishing, managing, and terminating communication sessions. |
| Service | Functionality provided by one protocol layer to the layer above it. |
| Service Model | Description of the services a layer offers while hiding implementation details. |
| Abstraction | Hiding internal implementation details while exposing only necessary functionality. |
| Interoperability | Ability of devices and software from different vendors to communicate successfully. |
| Protocol | Set of rules governing communication between peer entities in a network. |
