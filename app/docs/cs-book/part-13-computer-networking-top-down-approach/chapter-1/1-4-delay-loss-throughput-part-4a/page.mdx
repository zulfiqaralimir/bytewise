# Section 1.4 Delay, Loss, and Throughput in Packet-Switched Networks (Part 4A)

# Overview of Delay in Packet-Switched Networks

Up to this point in Chapter 1, we have learned:

- How the Internet is organized.
- What the Network Edge and Network Core are.
- How Packet Switching works.
- How routers forward packets.
- How packets travel from one network to another.

Now another important question arises:

> <span style={{ color: "#f97316", fontWeight: 700 }}>Why doesn't data reach its destination instantly?</span>

For example,

Suppose you click on a website.

```text
https://www.google.com
```

Does the webpage appear immediately?

No.

Even on a very fast Internet connection, there is always a small delay.

Sometimes the delay is:

```text
5 milliseconds
```

Sometimes:

```text
100 milliseconds
```

Sometimes:

```text
Several seconds
```

Why?

This section answers that question.

---

# Why Study Delay?

Every network engineer asks one important question:

> <span style={{ color: "#a78bfa", fontWeight: 700 }}>How fast can data travel through a network?</span>

To answer this question, we must understand:

- <span style={{ color: "#22c55e", fontWeight: 700 }}>Delay</span>
- <span style={{ color: "#22c55e", fontWeight: 700 }}>Packet Loss</span>
- <span style={{ color: "#22c55e", fontWeight: 700 }}>Throughput</span>

These three concepts determine the overall performance of a computer network.

---

# What Is Delay?

Delay refers to:

> **The amount of time required for a packet to travel from one point in the network to another.**

Another name for delay is:

> **Latency**

Although the words "delay" and "latency" are often used interchangeably, this book primarily uses the term **delay**.

# Why Can't Packets Travel Instantly?

Many beginners think:

> "Electricity travels at the speed of light."

So,

why isn't Internet communication instantaneous?

The answer is:

A packet must perform several tasks before reaching its destination.

Examples:

- <span style={{ color: "#f59e0b", fontWeight: 700 }}>Router processing</span>
- <span style={{ color: "#f59e0b", fontWeight: 700 }}>Waiting in queues</span>
- <span style={{ color: "#f59e0b", fontWeight: 700 }}>Transmission onto communication links</span>
- <span style={{ color: "#f59e0b", fontWeight: 700 }}>Physical signal propagation</span>

Each contributes to the total delay.

# Nodal Delay

The delay experienced at **one router (or one node)** is called:

> **Nodal Delay**

Definition:

> **Nodal Delay is the total delay experienced by a packet at a single network node.**

Every router introduces its own nodal delay.

---

# What Is a Node?

In networking,

a <span style={{ color: "#ec4899", fontWeight: 700 }}>**Node** is any device that can send, receive, or forward packets.</span>

Examples include:

- Computers
- Routers
- Switches
- Servers
- Access Points

In Section 1.4,

we mainly focus on routers.

---

# Components of Nodal Delay

<span style={{ color: "#06b6d4", fontWeight: 700 }}>every nodal delay consists of four components.</span>

```text

```

These four delays together form:

> **Total Nodal Delay**

---

# The Four Types of Delay

Throughout this section,

we will study:

## 1. Processing Delay

Time required by the router to examine the packet.

---

## 2. Queueing Delay

<span style={{ color: "#38bdf8", fontWeight: 700 }}>Time spent waiting inside the router's buffer.</span>

---

## 3. Transmission Delay

<span style={{ color: "#fbbf24", fontWeight: 700 }}>Time required to place all bits onto the communication link.</span>

---

## 4. Propagation Delay

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Time required for the signal to travel through the communication medium.</span>

# Mathematical Representation

Kurose represents total nodal delay as:

```text
d_nodal

=

d_proc

+

d_queue

+

d_trans

+

d_prop
```

where:

| Symbol | Meaning |
|----------|----------|
| d_proc | Processing Delay |
| d_queue | Queueing Delay |
| d_trans | Transmission Delay |
| d_prop | Propagation Delay |

<span style={{ color: "#eab308", fontWeight: 700 }}>This is one of the most important equations in introductory networking.</span>

---

# Understanding Each Delay

Although all four delays contribute to total communication time,

they originate from <span style={{ color: "#d946ef", fontWeight: 700 }}>different causes.</span>

---

## Processing Delay

Occurs because:

<span style={{ color: "#0ea5e9", fontWeight: 700 }}>The router must examine the packet.</span>

Example tasks:

- Check packet header.
- Verify integrity.
- Determine destination.
- Perform forwarding table lookup.

---

## Queueing Delay

Occurs because:

Other packets are already waiting.

The packet joins the queue.

Example:

```text
Packet A

Packet B

Packet C

â†“

Router
```

<span style={{ color: "#84cc16", fontWeight: 700 }}>Only one packet may be transmitted at a time.</span>

<span style={{ color: "#fb7185", fontWeight: 700 }}>The remaining packets wait.</span>

---

## Transmission Delay

Occurs because:

<span style={{ color: "#c084fc", fontWeight: 700 }}>A packet contains many bits.</span>

<span style={{ color: "#60a5fa", fontWeight: 700 }}>The router cannot place all bits onto the link instantly.</span>

Instead,

<span style={{ color: "#e879f9", fontWeight: 700 }}>bits leave one after another.</span>

---

## Propagation Delay

Occurs because:

Signals require time to travel through <span style={{ color: "#fde047", fontWeight: 700 }}>physical media.</span>

Even light traveling through optical fiber requires time.

Nothing travels infinitely fast.

---

# Delay Accumulates

Suppose communication passes through:

```text
Router A

â†“

Router B

â†“

Router C
```

Every router contributes:

```text
Processing Delay

+

Queueing Delay

+

Transmission Delay

+

Propagation Delay
```

Therefore,

the total communication delay becomes the sum of delays across all routers.

---

# Delay Is Not Constant

An important observation:

Not all delays behave the same way.

Some delays remain almost constant.

Others change continuously.

For example:

### Processing Delay

<span style={{ color: "#fca5a5", fontWeight: 700 }}>Usually very small and predictable.</span>

---

### Queueing Delay

Can vary from:

```text
0 ms
```

to

```text
Several Seconds
```

depending on traffic.

---

### Transmission Delay

Depends on:

- Packet size
- Link bandwidth

---

### Propagation Delay

Depends on:

- Distance
- Signal propagation speed

Delay affects nearly every Internet application.

# Delay vs Speed

<span style={{ color: "#fdba74", fontWeight: 700 }}>Many people believe:</span>

> <span style={{ color: "#86efac", fontWeight: 700 }}>"Higher Internet speed means zero delay."</span>

This is incorrect.

Increasing bandwidth reduces:

- Transmission Delay

However,

it does **not eliminate**:

- Processing Delay
- Queueing Delay
- Propagation Delay

<span style={{ color: "#a5b4fc", fontWeight: 700 }}>Even a 1 Gbps connection experiences delay.</span>

# Key Takeaways

- Delay is the time required for packets to travel through a network.
- Every router introduces nodal delay.
- Nodal delay consists of four components:
  - Processing Delay
  - Queueing Delay
  - Transmission Delay
  - Propagation Delay
- These four delays together determine overall network performance.
- End-to-end communication delay is the sum of delays across multiple routers.
- Different applications are affected differently by delay.

# Technical Vocabulary

| Term | Definition |
|------|------------|
| Delay | Time required for a packet to travel through a network. |
| Latency | Another term commonly used for network delay. |
| Node | Any device capable of sending, receiving, or forwarding packets. |
| Router | A network device that forwards packets between networks. |
| Nodal Delay | Total delay experienced by a packet at a single network node. |
| Processing Delay (`d_proc`) | Time required for a router to examine and process a packet. |
| Queueing Delay (`d_queue`) | Time a packet waits in a router's buffer before transmission. |
| Transmission Delay (`d_trans`) | Time required to place all packet bits onto a communication link. |
| Propagation Delay (`d_prop`) | Time required for a signal to travel across the communication medium. |
| End-to-End Delay | Total delay experienced by a packet from source to destination. |
