# Chapter 1: Computer Networks and the Internet

# 1.3 The Network Core

In the previous section, we learned about the **Network Edge**, where users and applications interact with the Internet. We saw that end systems such as laptops, smartphones, web servers, and cloud servers generate and receive data.

Now an important question arises:

> <span style={{ color: "#f97316", fontWeight: 700 }}>Once a packet leaves your computer, how does it actually travel across the Internet?</span>

The answer lies in another major component of the Internet:

> **The Network Core**

The Network Core is responsible for moving packets from one end system to another, even if they are thousands of kilometers apart.

---

# Learning Objectives

After completing this section, you should be able to:

- Explain what the Network Core is.
- <span style={{ color: "#10b981", fontWeight: 700 }}>Differentiate between the Network Edge and the Network Core.</span>
- <span style={{ color: "#a78bfa", fontWeight: 700 }}>Understand Packet Switching.</span>
- Explain <span style={{ color: "#f59e0b", fontWeight: 700 }}>why Packet Switching is used instead of sending an entire file at once.</span>
- Understand <span style={{ color: "#ef4444", fontWeight: 700 }}>how routers cooperate to deliver packets.</span>
- Follow the complete journey of a packet across the Internet.

---

# Internet Architecture

The Internet can be divided into two major parts.

```text
                 Internet

          ┌──────────────────┐
          │   Network Edge   │
          └──────────────────┘
                    │
                    │
                    ▼
          ┌──────────────────┐
          │   Network Core   │
          └──────────────────┘
                    │
                    │
                    ▼
          ┌──────────────────┐
          │   Network Edge   │
          └──────────────────┘
```

Notice that communication always follows this pattern:

```text
Sender
   ↓
Network Edge
   ↓
Network Core
   ↓
Network Edge
   ↓
Receiver
```

---

# What Is the Network Core?

The **Network Core** is <span style={{ color: "#14b8a6", fontWeight: 700 }}>the collection of interconnected routers and communication links that transport packets between end systems.</span>

Unlike the Network Edge, the Network Core does **not** run applications such as browsers or email clients.

Instead, its primary responsibility is:

- Receiving packets
- <span style={{ color: "#3b82f6", fontWeight: 700 }}>Examining destination addresses</span>
- <span style={{ color: "#ec4899", fontWeight: 700 }}>Determining the next path</span>
- Forwarding packets toward their destination

In simple words:

> The Network Core <span style={{ color: "#06b6d4", fontWeight: 700 }}>acts as the transportation system</span> of the Internet.

---

# Real-World Analogy

Imagine sending a parcel through a <span style={{ color: "#eab308", fontWeight: 700 }}>courier company.</span>

```text
Your House
     ↓
Local Courier Office
     ↓
Regional Distribution Center
     ↓
National Distribution Center
     ↓
Destination Distribution Center
     ↓
Friend's House
```

Your parcel passes through many distribution centers before reaching its destination.

The Internet works in exactly the same way.

Instead of parcels:

- We send packets.

Instead of courier centers:

- We use routers.

---

# Components of the Network Core

The Network Core mainly consists of:

## Routers

Routers are specialized networking devices.

Their job is to:

- Receive packets
- <span style={{ color: "#22c55e", fontWeight: 700 }}>Read destination addresses</span>
- <span style={{ color: "#8b5cf6", fontWeight: 700 }}>Choose the best outgoing path</span>
- Forward packets

Every packet passes through multiple routers before reaching its destination.

---

## Communication Links

Routers are connected using high-speed communication links.

Examples include:

- Fiber-optic cables
- High-speed <span style={{ color: "#d946ef", fontWeight: 700 }}>Ethernet</span>
- Optical backbone links
- Submarine fiber cables

These links allow routers to exchange packets extremely quickly.

---

# Example of a Simple Network Core

```text
Computer A
      │
      ▼
  Router 1
      │
      ▼
  Router 2
      │
      ▼
  Router 3
      │
      ▼
Computer B
```

<span style={{ color: "#f43f5e", fontWeight: 700 }}>Every router forwards packets closer to the destination.</span>

---

# Why Do We Need the Network Core?

Suppose your computer wants to communicate with Google's servers.

```text
Pakistan
      │
      │
Google Data Center (USA)
```

The distance may be over 12,000 kilometers.

Obviously:

- There is no direct cable from your laptop to Google.

Instead, packets travel through many routers connected across different networks.

The Network Core makes long-distance communication possible.

---

# Main Job of the Network Core

The Network Core has only one fundamental responsibility:

> **Move packets from the source host to the destination host.**

Everything else—<span style={{ color: "#0ea5e9", fontWeight: 700 }}>routing, forwarding, switching</span>—is done to achieve this goal.

---

# Network Edge vs Network Core

| Network Edge | Network Core |
|--------------|--------------|
| Contains end systems | Contains routers |
| Runs applications | Forwards packets |
| Generates data | Moves data |
| Receives data | Routes data |
| <span style={{ color: "#84cc16", fontWeight: 700 }}>Users interact here</span> | <span style={{ color: "#fb7185", fontWeight: 700 }}>Routers operate here</span> |

---

# 1.3.1 Packet Switching

The Internet uses a communication technique called:

> **Packet Switching**

Packet Switching is one of the most important concepts in computer networking.

Almost every modern network—including the Internet—uses Packet Switching.

---

# What Is Packet Switching?

Instead of sending one huge block of data, the sender divides the data into many smaller pieces called:

> **Packets**

Each packet <span style={{ color: "#c084fc", fontWeight: 700 }}>travels independently</span> through the network.

At the destination, all packets are reassembled to reconstruct the original data.

---

# Example

Suppose you want to send a:

```text
100 MB Video
```

The Internet does **not** send:

```text
100 MB
```

as one giant transmission.

Instead:

```text
100 MB Video

        ↓

Packet 1
Packet 2
Packet 3
Packet 4
...
Packet N
```

Thousands of packets are created.

Each packet travels separately.

---

# Why Split Data into Packets?

Imagine a highway.

Suppose one truck carries an entire factory.

The truck would:

- Occupy the road for hours.
- Block all other traffic.
- Cause congestion.

Instead, factories ship goods using many smaller trucks.

Advantages:

- Better traffic flow.
- Easier routing.
- Faster delivery.
- Multiple trucks can use different roads.

Packet Switching follows exactly the same idea.

---

# Characteristics of Packet Switching

Packet Switching has several important properties.

## Data Is Divided

Large files become many packets.

---

## Packets Are Independent

Every packet is treated separately.

Different packets may even travel along different routes.

---

## Shared Resources

Communication links are shared by many users simultaneously.

No user permanently owns a communication link.

---

## Efficient Utilization

Bandwidth is used only when data is actually being transmitted.

Idle users do not waste network resources.

---

# Packet Structure

Every packet generally contains two parts.

```text
┌────────────────────────────┐
│ Header                     │
├────────────────────────────┤
│ Payload (Actual Data)       │
└────────────────────────────┘
```

---

## Header

The header contains control information.

Examples include:

- Source Address
- Destination Address
- Packet Length
- Protocol Information
- Error Detection Information

The header helps routers determine how to forward the packet.

---

## Payload

The payload contains:

> The actual user data.

Examples:

- Email content
- Image data
- Video data
- Web page data

---

# Why Is Packet Switching So Popular?

Packet Switching offers many advantages.

## Efficient Bandwidth Utilization

Communication links are shared dynamically.

Bandwidth is not reserved unnecessarily.

---

## Supports Millions of Users

Multiple users can transmit simultaneously.

The network automatically shares available resources.

---

## High Reliability

If one path fails:

Packets can often be rerouted through another path.

---

## Scalability

Packet Switching allows the Internet to grow from:

- Thousands of computers

to

- Billions of connected devices.

---

## Cost Effective

Resources are shared among many users.

This reduces infrastructure costs.

---

# Important Observation

Packet Switching does **not** guarantee that every packet follows the same path.

Example:

```text
Packet 1
      ↓
Router A
      ↓
Router B

Packet 2
      ↓
Router C
      ↓
Router D
```

Both packets may still reach the same destination.

The destination computer reorders packets if necessary.

---

# Store-and-Forward Transmission

In Packet Switching, when a router receives a packet, it does **not** immediately start forwarding it.

A router must first receive the **entire packet** before forwarding it to the next router.

This process is called:

> **Store-and-Forward Transmission**

It is one of the most fundamental concepts in computer networking.

---

# What Does "Store-and-Forward" Mean?

## Step 1: Store

The router first stores the **complete packet** in its memory (buffer).

---

## Step 2: Forward

After the complete packet has arrived, the router examines the packet header, determines the appropriate outgoing link, and forwards the packet.

Therefore:

```text
Receive Entire Packet
        ↓
Store Packet
        ↓
Read Header
        ↓
Choose Outgoing Link
        ↓
Forward Packet
```

---

# Why Doesn't the Router Forward Immediately?

Imagine receiving a courier parcel.

Suppose someone hands you only half of the parcel.

Can you immediately deliver it?

Of course not.

You must first receive the **entire parcel** before sending it to someone else.

Routers work exactly the same way.

They must receive the complete packet before forwarding it.

---

# Why Is Store-and-Forward Necessary?

Routers need the complete packet because they must:

- Verify the packet is complete.
- Read the destination address.
- Check for transmission errors.
- Decide the next outgoing link.

Without the complete packet, the router cannot safely forward data.

---

# Visualizing Store-and-Forward

Imagine the following network:

```text
Host A
   │
   ▼
Router R1
   │
   ▼
Router R2
   │
   ▼
Host B
```

Suppose Host A sends one packet.

### Step 1

```text
Host A
   │████████████│
   ▼
Router R1
```

Router R1 receives the entire packet.

---

### Step 2

```text
Router R1
   │████████████│
   ▼
Router R2
```

Only now does Router R1 transmit the packet.

---

### Step 3

```text
Router R2
   │████████████│
   ▼
Host B
```

Again, Router R2 waits until it has received the entire packet before forwarding it.

---

# What Is a Hop?

Whenever a packet moves from one router to the next, it completes one:

> **Hop**

Example:

```text
Host
   │
Router A
   │
Router B
   │
Router C
   │
Server
```

Here we have:

- Hop 1
- Hop 2
- Hop 3

Each hop requires store-and-forward transmission.

---

# Transmission Delay

Because every router waits for the complete packet, every hop introduces some delay.

This delay is called:

> **Transmission Delay**

Transmission Delay is:

> The time required to place all bits of a packet onto the communication link.

Notice something important.

Transmission Delay is **not** the time required for the signal to travel.

It is the time needed to transmit every bit.

Think of filling a pipeline with water.

The water cannot leave the pipeline until enough water has entered.

---

# Transmission Delay Formula

```text
Transmission Delay = Packet Length / Transmission Rate
```

or

```text
d_trans = L / R
```

where:

| Symbol | Meaning |
|----------|---------|
| L | Packet Length (bits) |
| R | Transmission Rate (bits/sec) |

---

# Example 1

Packet Length:

```text
8,000 bits
```

Transmission Rate:

```text
2,000 bits/sec
```

Then:

```text
Transmission Delay = 8000 / 2000 = 4 seconds
```

---

# Example 2

Packet Size:

```text
10,000 bits
```

Link Speed:

```text
1 Mbps = 1,000,000 bits/sec
```

Transmission Delay:

```text
10,000 / 1,000,000 = 0.01 seconds = 10 milliseconds
```

---

# What Affects Transmission Delay?

## Directly Proportional to Packet Length

More bits → More transmission time.

## Inversely Proportional to Transmission Rate

Higher bandwidth → Less transmission time.

---

# Multiple Routers

Suppose a packet travels through three routers, and each link has:

```text
Transmission Delay = 5 ms
```

```text
Host → R1 → R2 → R3 → Server
```

Total transmission delay:

```text
5 + 5 + 5 + 5 = 20 ms
```

Every communication link contributes its own transmission delay.

---

# Queueing and Queueing Delay

Another important question:

> **What happens if packets arrive faster than the router can forward them?**

The answer: they wait.

---

# What Is Queueing?

A router can process only a limited number of packets every second.

If packets arrive faster than they can be transmitted, they wait in a:

## Queue

---

# Queue Inside a Router

A router contains memory called a:

## Buffer

Packets are temporarily stored inside this buffer before being forwarded.

```text
Incoming Packets

Packet A
Packet B
Packet C
Packet D

      │
      ▼

+------------------------+
|        BUFFER          |
|------------------------|
| Packet A               |
| Packet B               |
| Packet C               |
| Packet D               |
+------------------------+

      │
      ▼

Outgoing Link
```

Only one packet can usually be transmitted at a time. The others wait.

---

# Queueing Delay

Packets inside the queue cannot move immediately.

This waiting time is called:

## Queueing Delay

> The time a packet spends waiting inside a router's buffer before transmission.

Unlike Transmission Delay, Queueing Delay is **not fixed**. It depends on:

- Current traffic
- Number of arriving packets
- Router workload
- Available bandwidth

---

# Example

A router can send 1 packet per second.

Five packets arrive simultaneously.

| Packet | Waits | Sent at Second |
|--------|-------|---------------|
| A | 0 sec | 1 |
| B | 1 sec | 2 |
| C | 2 sec | 3 |
| D | 3 sec | 4 |
| E | 4 sec | 5 |

Packet E experienced a Queueing Delay of 4 seconds.

---

# Congestion

When too many packets compete for limited network resources, the network experiences:

## Congestion

> Congestion occurs when packet arrival rate exceeds the network's forwarding capacity.

Causes of Congestion:

- Many users online simultaneously
- Limited bandwidth on a link
- Large data transfers (video streaming, cloud backups)
- Slow outgoing links

Consequences:

- Longer Queueing Delay
- Increased Latency
- Reduced Throughput
- Packet Loss

---

# Packet Loss

Buffers are not infinite — they have limited storage.

Suppose the buffer is full:

```text
Buffer FULL

↓

Another packet arrives

↓

No space available

↓

Router drops the packet
```

This is called:

## Packet Loss

Different applications react differently to packet loss:

| Application | Effect |
|-------------|--------|
| Email | Can retransmit |
| Web Browsing | Can retransmit |
| Video Streaming | May skip frames |
| Voice Calls | Broken audio |
| Online Gaming | Lag |

---

# Relationship Between These Concepts

```text
Many Packets Arrive

↓

Queue Forms

↓

Queueing Delay Increases

↓

Buffer Becomes Full

↓

Packet Loss Occurs

↓

Congestion Becomes Worse
```

Understanding this chain is extremely important because later chapters discuss:

- TCP Congestion Control
- Active Queue Management
- RED (Random Early Detection)
- Quality of Service (QoS)

---

# Forwarding Tables

Another important question:

> **How does a router know where to send a packet?**

Every router maintains a table called the:

## Forwarding Table

A Forwarding Table tells the router:

> If a packet is destined for a particular network, which outgoing link should be used?

Think of it as a GPS navigation system — it examines your destination and chooses the best road, step by step.

---

# Forwarding Table Example

| Destination Network | Outgoing Link |
|---------------------|---------------|
| Network A | Link 1 |
| Network B | Link 2 |
| Network C | Link 3 |
| Default | Link 4 |

When a packet arrives destined for **Network B**:

```text
Read Destination

↓

Search Forwarding Table

↓

Network B → Link 2

↓

Packet leaves through Link 2
```

---

# Step-by-Step Packet Forwarding

```text
Packet Arrives

↓

Read Header

↓

Extract Destination Address

↓

Search Forwarding Table

↓

Find Matching Entry

↓

Select Outgoing Link

↓

Forward Packet
```

This process takes only microseconds in modern routers.

---

# The Next Hop Concept

Routers usually do **not** know the complete path to the destination.

Instead, they know only where to send the packet next.

This is called the:

## Next Hop

```text
Router A knows only:

Next Hop → Router B
```

Router B decides the next hop, and so on until the packet reaches its destination.

---

# Why Use Next-Hop Forwarding?

Imagine driving across an unfamiliar country.

Instead of memorizing every road:

Your GPS tells you only the next turn.

Networking follows the same principle.

Each router only decides: **Where should the packet go next?**

---

# Complete Packet Journey

```text
Browser
   ↓
Operating System creates packets
   ↓
Packets leave the laptop
   ↓
Home Router
   ↓
ISP Router
   ↓
Regional Router
   ↓
Backbone Router
   ↓
Google Router
   ↓
Google Server
```

Every router repeats exactly the same forwarding algorithm:

```text
Receive Packet
       ↓
Store Packet
       ↓
Read Header
       ↓
Extract Destination IP
       ↓
Consult Forwarding Table
       ↓
Choose Next Hop
       ↓
Queue (if necessary)
       ↓
Transmit Packet
```

---

# Common Misconceptions

## Misconception 1

> Routers read the entire packet.

**Incorrect.** Routers primarily inspect the **packet header**, especially the destination IP address. The payload remains untouched.

---

## Misconception 2

> Every packet follows the same path.

**Incorrect.** Packets belonging to the same file may travel through different routes.

---

## Misconception 3

> Routers know the complete path to every destination.

**Incorrect.** Routers generally know only the **next hop**.

---

## Misconception 4

> Packet Switching guarantees zero delay.

**Incorrect.** Packets may experience Transmission Delay, Queueing Delay, Processing Delay, and Propagation Delay.

---

# Key Takeaways

- The Network Core is responsible for transporting packets across the Internet.
- Routers are the primary devices inside the Network Core.
- The Internet uses Packet Switching — large messages are divided into packets.
- Every packet is forwarded independently by routers.
- Routers use Store-and-Forward Transmission — the complete packet must arrive before forwarding.
- Every hop introduces Transmission Delay (d_trans = L / R).
- Packets waiting in a buffer experience Queueing Delay.
- Heavy traffic causes congestion and eventually Packet Loss.
- Forwarding Tables map destination addresses to outgoing links.
- Each router chooses only the next hop — it does not know the full path.
- Packet Switching allows efficient sharing of network resources for billions of users.
